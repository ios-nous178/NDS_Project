시나리오 보드 ↔ 멀티스크린 스위처 양방향 동기화 — 목업별 브릿지 스크립트 불필요

목업의 상단 화면 스위처(`.mockup-screen[data-active]`)와 빌드가 주입하는 시나리오 보드가 화면-가시성 계약이 서로 달라(보드 옵저버는 `hidden/style/class` 만 보고 스위처는 `data-active` 로 전환), 탭을 바꿔도 보드가 안 따라오고 보드 단계를 클릭해도 화면이 안 바뀌었습니다. 둘을 `nds-scenario-nav` 한 이벤트로 묶어 DS 차원에서 양방향 동기화되게 고쳤습니다.

- **탭 전환 → 보드** 의 '지금 보는 화면'이 따라옵니다(스위처가 활성화 시 이벤트 발신 + 보드 옵저버가 `data-active` 관찰).
- **보드 단계 클릭 → 실제 화면 전환**(+ 상단 탭 동기화). 스위처가 이벤트를 수신해 해당 화면을 활성화.
- 각 `.mockup-screen` 에 `data-screen="<키>"` 만 달면 자동 동작 — **MutationObserver/이벤트 브릿지 스크립트를 손으로 끼울 필요가 없습니다**(multi-screen 가이드에 명시).
