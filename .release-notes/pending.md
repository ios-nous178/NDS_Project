🚨 *업데이트 이후 Claude Desktop 꼭 종료 후 다시 실행!!!* 🚨
새 MCP 가이드/카탈로그가 적용되려면 Claude Desktop 을 완전히 종료(⌘Q)했다가 다시 켜 주세요. 그냥 창만 닫으면 옛 버전이 그대로 떠 있어요.

*배경*

브랜드가 4개에서 5개로 늘었고, 그동안 패키지·자산·토큰 이름이 조금씩 어긋나 있던 부분을 한 번에 정리한 큰 업데이트예요. 새 브랜드 합류와 함께 "디자인 시스템 전체의 이름·자산·규칙을 하나의 기준으로 통일" 하는 데 초점을 맞췄습니다.

*주요 변경사항*

*1. 5번째 브랜드 Runmile 합류*
· 러닝/마라톤 브랜드 Runmile 신설 — 색·여백 토큰, 전용 아이콘 80여 종, 하단 네비게이션, 스토리북 데모까지 한 번에 도입.
· 이제 지원 브랜드는 5종 (Trost / Geniet / NudgeEAP / CashwalkBiz / Runmile).

*2. 자산 한곳 모으기 — 브랜드 로고·일러스트 통합*
· 브랜드 로고(5종) · SNS 로그인 로고(네이버/카카오/구글/애플) · 프로필 이미지 · 빈화면·에러 일러스트 · 마라톤 행사 그림까지 *총 67종 자산을 DS 안에 내장*.
· 각 서비스가 따로 자산을 들고 다닐 필요 없이 DS 에서 바로 꺼내 쓸 수 있어요. 곳곳에 중복으로 복사돼 있던 로고 파일도 정리.

*3. 통합 미디어 박스 — Asset 컴포넌트*
· 이미지·아이콘·이니셜·Lottie 를 *한 컴포넌트로 통합* 한 `Asset` 도입. 카드 썸네일, 카테고리 시그니처, 상품 이미지, 채팅 첨부 같은 *사람이 아닌 일반 미디어 박스* 에 사용.
· 모양(사각/라운드/원형)·크기·우측 하단 상태 점·옆 자산과 겹치기까지 한 묶음으로 일관 적용.
· 사람 프로필은 기존 Avatar 그대로, 일반 미디어는 Asset 으로 역할 분리.

*4. 어드민 화면 레이아웃 표준화*
· 어드민/CMS 화면마다 매번 수백 줄씩 다시 짜던 페이지 골격(상단바·섹션·폼 줄)을 *정해진 레이아웃 한 세트* 로 흡수.
· 화면마다 들쭉날쭉하던 여백·정렬이 자동으로 같아지고, 직접 CSS 를 짜다 생기던 흔들림을 차단.

*5. 브랜드 × 컴포넌트 커버리지 페이지*
· 어떤 브랜드가 어떤 컴포넌트를 어디까지 지원하는지 *한 페이지에서 한눈에* 확인 (코드+Figma 둘 다 ✓ / 코드만 / 미구현 3단계 표시).

*6. 시멘틱 토큰 이름 통일 — `--semantic-*` 이 색·여백 모두 흡수*
· 색은 `--semantic-`, 여백은 `--gap-`/`--inset-` 로 갈려 있던 비일관 정리. 이제 `--semantic-` 이 보이면 *Figma 가 기준*, `--nds-` 가 보이면 *DS 자체 슬롯* 으로 한눈에 구분.
· 옛 이름(`--gap-*` / `--inset-*`)은 *그대로 동작* (새 이름의 별칭으로 함께 제공) — 기존 코드 안 깨져요. 새 코드만 `--semantic-` 권장.

*7. 패키지·브랜드 이름 정리*
· 패키지 이름을 `@nudge-eap/*` → `@nudge-design/*` 로 통일 (DS 를 npm 으로 쓰는 쪽은 import 경로 변경 필요).
· 브랜드 코드명 "캐포비" 를 영문 정식 표기 *CashwalkBiz* 로 통일 — 다른 브랜드(Trost/Geniet/NudgeEAP)와 표기 기준 일치. 화면 표시명 "Cashwalk for Business" 와 한국어 "캐포비" 는 그대로.

*8. 결함 수정 · 가이드 강화*
· 탭을 눌러도 선택이 안 바뀌던 결함 수정 (HTML 탭).
· 비공개 저장소에서도 문서 "마크다운으로 열기" 가 정상 동작.
· 버튼 라벨 두 줄 줄바꿈 방지, 입력 도움말/에러 동시 노출 금지 등 브랜드 가이드 규칙 보강.

*기대효과*
· 새 브랜드도 같은 토큰·자산·규칙 위에서 바로 시작 — 브랜드가 늘어도 일관성 유지.
· 로고·일러스트·프로필을 DS 한곳에서 공급 → 서비스마다 자산 챙기는 수고 제거.
· 토큰·패키지·브랜드 이름이 한 기준으로 정리돼 "이게 Figma 기준인지 DS 자체인지" 헷갈릴 일이 줄어듭니다.
