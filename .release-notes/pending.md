🚨 *업데이트 이후 Claude Desktop 꼭 종료 후 다시 실행!!!* 🚨
새 MCP 가이드/카탈로그가 적용되려면 Claude Desktop 을 완전히 종료(⌘Q)했다가 다시 켜 주세요. 그냥 창만 닫으면 옛 버전이 그대로 떠 있어요.

*배경*

브랜드가 4개에서 5개로 늘었고, 그동안 흩어져 있던 패키지·자산·토큰 이름을 한 기준으로 정리하면서, 입력 폼에 쓸 컴포넌트와 버튼 체계까지 크게 보강한 큰 업데이트예요. "새 브랜드 합류 + 전체 규칙 통일 + 폼/버튼 경험 강화" 가 핵심입니다.

*주요 변경사항*

*1. 5번째 브랜드 Runmile 합류*
· 러닝/마라톤 브랜드 Runmile 신설 — 색·여백 토큰, 전용 아이콘 80여 종, 하단 네비게이션, 스토리북 데모까지 한 번에 도입.
· 이제 지원 브랜드는 5종 (Trost / Geniet / NudgeEAP / CashwalkBiz / Runmile).

*2. 자산 한곳 모으기 — 브랜드 로고·일러스트 통합*
· 브랜드 로고(5종) · SNS 로그인 로고(네이버/카카오/구글/애플) · 프로필 이미지 · 빈화면·에러 일러스트 · 마라톤 행사 그림까지 *총 67종 자산을 DS 안에 내장*.
· 각 서비스가 따로 자산을 들고 다닐 필요 없이 DS 에서 바로 꺼내 쓸 수 있어요. 중복 복사돼 있던 로고 파일도 정리.

*3. 새 입력·폼 컴포넌트 대거 합류*
· *인증번호 입력(OtpInput)* — 자리별 칸 → 한 줄 입력 + 우측 카운트다운 타이머·재전송형으로 전환(실제 서비스 화면과 일치).
· *휴대폰 번호 입력(PhoneInput)* — 국가코드 드롭다운과 번호칸 분리 + 입력하면 하이픈(010-1234-5678) 자동.
· *인라인 안내·주의·에러 박스(NoticeAlert)* · *주소 검색→선택→상세(AddressPicker)* · *지역 선택 트리* · *동의 체크리스트* · *펼침형 표(DataTable)* 등 폼/리스트 컴포넌트 합류.
· 비밀번호 보기 토글, 글자수 카운터 같은 입력 공통 기능도 표준화.

*4. 버튼 체계 단일화*
· 버튼을 *모양(기본/캡슐) × 강조(채움 Solid·연한 Soft·테두리 Outlined) × 톤(Primary·Secondary·Neutral)* 세 축으로 정리.
· 이름만 다르고 거의 같던 중복 변형을 정리해, 어떤 버튼을 골라야 할지가 단순해졌어요. 밝은 회색 버튼에서 흰 글자가 안 보이던 가독성 문제도 수정.

*5. 캐시워크 for Business 브랜드 정밀 정합*
· 캐포비 시그니처 *검정/회색 CTA 를 brand 규칙대로 자동 적용* (색을 화면마다 직접 박지 않아도 됨).
· 어드민 *폼·표·페이지 패턴* 정비 + 입력칸 높이/너비 표준화 + 신규 *ConfirmTooltip(확인 말풍선)*.
· 캐포비 화면에서 쓰면 안 되는 톤/조합은 *제작 단계에서 자동 경고* 되도록 가드 추가.

*6. 통합 미디어 박스 — Asset 컴포넌트*
· 이미지·아이콘·이니셜·Lottie 를 *한 컴포넌트로 통합* 한 `Asset` 도입. 카드 썸네일·카테고리 시그니처·상품 이미지·채팅 첨부 같은 *사람이 아닌 일반 미디어 박스* 에 사용.
· 사람 프로필은 기존 Avatar 그대로, 일반 미디어는 Asset 으로 역할 분리.

*7. 어드민 화면 레이아웃 표준화*
· 어드민/CMS 화면마다 매번 다시 짜던 페이지 골격(상단바·섹션·폼 줄)을 *정해진 레이아웃 한 세트* 로 흡수 — 여백·정렬이 자동으로 같아집니다.

*8. 시멘틱 토큰·패키지·브랜드 이름 통일*
· 색은 `--semantic-`, DS 자체 슬롯은 `--nds-` 로 한눈에 구분 (옛 `--gap-*`/`--inset-*` 는 별칭으로 그대로 동작 — 기존 코드 안 깨짐).
· 패키지 이름 `@nudge-eap/*` → `@nudge-design/*` 통일(npm 소비 측 import 경로 변경 필요) · 브랜드 코드명 *CashwalkBiz* 로 영문 표기 통일.
· *브랜드 × 컴포넌트 커버리지 페이지* — 어떤 브랜드가 뭘 어디까지 지원하는지 한 페이지에서 확인.

*9. 목업 품질·설계안 가드레일 강화*
· 복잡한 화면은 코드 전에 *한 장의 설계안(DesignSpec)* 을 먼저 만들어 동의받는 단계 도입 — 잘못된 브랜드명·하드코딩 색·없는 토큰을 미리 차단.
· 화면 제작 시 브랜드/접근성/구조 오류를 자동으로 잡는 검수 규칙 추가 + 탭 클릭 결함 등 결함 수정.

*기대효과*
· 폼 화면을 만들 때 골라 쓸 컴포넌트가 늘고, 브랜드별 색·버튼이 자동으로 맞아 손이 덜 갑니다.
· 캐시워크 for Business 어드민/서비스 화면의 톤이 한층 일관돼집니다.
· 토큰·패키지·브랜드 이름이 한 기준으로 정리돼 "Figma 기준인지 DS 자체인지" 헷갈릴 일이 줄어듭니다.
