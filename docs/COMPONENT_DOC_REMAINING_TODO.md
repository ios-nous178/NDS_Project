## 1. Variant/Size별 실제 서비스 시나리오 매핑

현재 각 컴포넌트의 variant/size 테이블에 "주요 CTA, 강한 강조" 같은 **일반적 사용 시점**은 있지만, **실제 서비스 화면 기준 구체적 시나리오**는 없습니다.

### 목표

각 컴포넌트 문서의 variant/size 테이블에 "서비스 적용 예시" 컬럼을 추가하거나, 별도 섹션으로 매핑합니다.

### 예시 (Button)

| Variant + Size                | 서비스 적용 예시                                   |
| ----------------------------- | -------------------------------------------------- |
| `solid` + `xl` + `primary`    | 상담 예약 완료, 회원가입 제출 등 페이지 최하단 CTA |
| `outlined` + `lg` + `primary` | CTA 옆 보조 버튼 (취소, 이전 등)                   |
| `soft` + `md`                 | 카드 내부 부드러운 액션 (상세 보기, 더 알아보기)   |
| `outlined-sub` + `sm`         | 리스트 항목 옆 부가 액션 (수정, 삭제)              |

### 대상 컴포넌트

- [ ] **Button** — variant(4) x size(6) x color(2) 조합 중 주요 패턴
- [ ] **Badge** — variant(6)별 사용 화면
- [ ] **Chip** — variant(3)별 사용 화면
- [ ] **Tabs** — variant(3: line/pill/square)별 사용 화면
- [ ] **Input** — size별 사용 화면
- [ ] **Modal / Popup / BottomSheet** — maxWidth 패턴별 적용 사례

### 작성 주체

기획 또는 디자인 담당자가 Figma 시안 기반으로 작성 → 개발 리뷰

### 2-3. AI 파이프라인 컨텍스트 범위 결정

AI가 디자인 시스템 문서를 활용할 때 어떤 범위를 입력으로 넣을지 결정이 필요합니다.

| 범위                      | 장점                     | 단점                        |
| ------------------------- | ------------------------ | --------------------------- |
| 단일 컴포넌트 문서        | 토큰 절약, 정확한 포커스 | 유사 컴포넌트 간 선택 불가  |
| overview + 해당 컴포넌트  | 선택 가이드 포함         | 토큰 소비 증가              |
| 전체 디자인 시스템 문서   | 가장 풍부한 컨텍스트     | 토큰 비용 높음, 노이즈 가능 |
| 문서 + 컴포넌트 소스 코드 | 구현 디테일까지 참조     | 가장 비용 높음              |
