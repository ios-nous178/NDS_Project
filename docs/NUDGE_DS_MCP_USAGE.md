## Quick start

부탁할 때 3가지만 챙기면 됩니다. Claude 가 카탈로그와 가이드를 알아서 호출해요.

1. **종류** — 트로스트 서비스 / 캐포비 어드민 / 일반 어드민 중 하나
2. **시각 자료 1-2장** — 정답 1장, 피하고 싶은 오답 1장 (스크린샷 또는 Figma 링크)
3. **"NDS 써서" 한 마디** — 수정·추가·검증 매 턴마다

세 가지가 갖춰지면 한 줄 부탁으로 시작할 수 있어요.

> "NDS 써서 트로스트 서비스 상담사 목록 모바일 만들어줘. (스크린샷 첨부)"

검증·빌드·사용 현황표 같은 후속 단계는 묶어서 부탁해도 됩니다.

> "NDS 써서 검증하고 HTML 한 파일로 뽑아줘."

## 시작 전 30초 체크

### 시안 폴더가 열려 있나요?

채팅창 하단 표시줄에 폴더 이름만 보이면 됩니다. git 브랜치나 워크트리 표시가 보이면 다른 폴더가 열린 상태예요. 거기서 시안을 만들면 그 폴더의 원본 코드와 섞입니다.

![좋은 작업 폴더 — 시안 전용 폴더(nudge_133)만 열린 상태. git 브랜치/워크트리 표시 없음](/img/install/workspace-good.png)

폴더 이름만 표시되면 OK.

![나쁜 작업 폴더 — NudgeEAPDesignSystem 같은 다른 폴더가 열림. main 브랜치 + 워크트리 표시](/img/install/workspace-bad.png)

git 브랜치나 워크트리 표시가 보이면 ⌘+O 로 시안 폴더를 다시 여세요.

### 커넥터가 켜져 있나요?

채팅창의 ➕ 메뉴 → 커넥터 를 열어 `Nudge Design System` 토글이 ON 인지 확인하세요.

![커넥터 확인 — ➕ 메뉴 → 커넥터 패널에 'Nudge Design System' 토글이 ON 상태](/img/install/connector-check.png)

또는 채팅창에 `/mcp` 를 입력하면 목록에 `nudge-ds` 가 ✓ 로 보입니다. 커넥터 UI 이름은 `Nudge Design System`, 기술 ID 는 `nudge-ds` — 같은 것을 두 가지로 부릅니다.

안 보이거나 꺼져 있으면 ⌘Q 로 완전히 종료하고 재시작하세요.

## 화면 종류 정하기

만들 화면 종류에 따라 라이브러리와 톤이 달라져요. 한 단어만 붙이면 충분합니다.

| 종류               | 라이브러리               | 톤                  |
| ------------------ | ------------------------ | ------------------- |
| 서비스 (사용자 앱) | Nudge DS                 | 브랜드, 모바일 우선 |
| **캐포비 어드민**  | Nudge DS (캐포비 브랜드) | 데스크톱, 폼 위주   |
| 일반 어드민        | Ant Design               | 운영툴, 데스크톱    |

캐포비는 어드민이지만 Ant Design 이 아닌 _캐포비 브랜드 DS_ 로 만들어요. "캐포비 어드민" 이라고 명시하면 일반 어드민과 자동으로 분리됩니다.

좋은 예시:

- "트로스트 서비스 마이페이지"
- "캐포비 어드민 회원 관리"
- "어드민 회원 관리" (= 일반 어드민)

종류가 모호하면 결과도 모호해져요. "회원 관리 페이지 만들어줘" 같은 부탁은 피해주세요.

## 부탁 패턴

### PRD + 최소 5요소

PRD 가 있으면 채팅창에 링크나 핵심 본문을 같이 던지세요. 의사결정 정확도가 크게 올라가요.

| 요소      | 예시                                    |
| --------- | --------------------------------------- |
| 종류      | 서비스 / 캐포비 어드민 / 일반 어드민    |
| 브랜드    | 트로스트 / 지니어트 / NudgeEAP / 캐포비 |
| 페이지    | 상담사 목록 / 회원 관리                 |
| 기기      | 모바일 / 데스크톱                       |
| 핵심 구성 | 검색바·필터·카드·페이지네이션 (위→아래) |

한 줄 부탁 예시:

> "트로스트 서비스 상담사 목록, 모바일. 검색바 + 카테고리 필터(불안/우울/관계) + 상담사 카드 + 페이지네이션."

## 자주 쓰는 부탁 예시

복붙해서 바로 쓰세요. 수정·추가·검증·빌드 단계에서는 "NDS 써서" 한 마디를 꼭 끼워주세요.

### 조사

- "탭(Tabs) 컴포넌트 사용법과 함정 알려줘."
- "이 스크린샷에 어떤 DS 컴포넌트 조합이 좋을지 추천. (이미지 첨부)"

### 시안 만들기

- "트로스트 서비스 상담사 목록, 모바일. 검색바 + 카테고리 칩 + 상담사 카드 + 페이지네이션."
- "캐포비 어드민 회원 관리, 데스크톱. 검색·필터 + 회원 테이블 + 우측 상단 '신규 등록' 버튼."

### 수정 · 추가

- "NDS 써서 상담사 카드에 평점·리뷰 수 추가. 별점 컴포넌트 있으면 그거 쓰고."
- "(스크린샷 첨부) NDS 토큰 다시 확인하고 이 빨간색 더 부드러운 톤으로."

### 검증

- "NDS 써서 검증 + 위반 자동 수정. 0건 될 때까지 2회 이상, 같은 위반 3회 연속이면 멈추고 보고."
- "NDS 의 CTA 그룹 가이드 보고 위계 봐줘. 대표 액션 1개만 강조."
- "NDS 로 이 화면 품질 점수 매겨줘." → `score_mockup_quality` 가 **코드 점수(D1: 색상·타이포·여백·레이아웃·컴포넌트·아이콘)** 와 **정성 점수(D2: UX·인터랙션·흐름·폼, 독립 채점)** 를 함께 내고 **통과 / 주의 / 미달**(≥80 / ≥60 / &lt;60) 판정을 줍니다. 데스크톱 Nudge Studio 가 빌드 후 보여주는 점수 카드와 같은 기준이에요. (정성 점수는 `claude` 가 있어야 채점되고, 없으면 코드 점수만 나옵니다.)

### 빌드 · 공유

- "NDS 써서 HTML 한 파일로 뽑아줘."
- "NDS 써서 사용 현황표 만들어줘. DS 사용 비율 한눈에."

### 가이드 · 회고

- "DS 금지 패턴 알려줘."
- "방금 만든 화면이 따른 가이드 / 패턴 알려줘."

## HTML 한 파일로 뽑기

**슬랙이나 메일로 공유하려면 이 단계가 꼭 필요해요.** dev 서버 URL 은 본인 PC 에서만 열리고, 일반 빌드 결과(`dist/` 폴더 통째) 는 파일이 여러 개라 첨부가 안 돼요. 한 파일로 묶어야 비로소 받는 분 PC 에서 더블클릭으로 열립니다.

> "NDS 써서 HTML 뽑아줘"

이 한 줄이면 클릭·탭 인터랙션, 아이콘, 스타일이 전부 inline 된 `dist/index.html` 한 파일이 생성됩니다. 슬랙에 드래그앤드롭, 메일에 첨부 — 그게 끝이에요.

빌드는 시작 전에 폴더 상태를 점검합니다. 아래 조건 중 하나라도 걸리면 거부 — 거부 메시지를 Claude 에 그대로 보여주면 자동으로 풀어줘요.

:::warning 빌드가 거부될 수 있는 경우

- **시각 자료가 없음** — 정답 화면 1-2장(스크린샷 / Figma 링크) 을 시안 폴더에 넣어주세요.
- **PRD / 기획 문서만 있음** — 텍스트 PRD 는 시각 자료가 아니에요. 한 장 더 필요합니다.
- **이전 화면 자료가 남아 있음** — 새 화면용으로 새로 모아 달라고 부탁하세요.
- **src 에 raw .html / 토큰 덮어쓴 CSS** — `.tsx` 로 옮기고 시멘틱 토큰으로 바꿔 달라고 부탁하세요.

:::

## Claude 가 자동으로 잡아주는 것

저장 시점에 색상값(hex → 토큰), 4pt grid, 그라데이션, DS 외부 컴포넌트 import, 이모지 사용 등을 자동으로 검증하고 위반을 자동 수정합니다. `Button` · `Card` · `Chip` · `Tabs` · `Select` · `Banner` · `Modal` 같은 핵심 컴포넌트의 함정도 자동으로 피해줘요.

검증은 **2회 반복이 룰**입니다. 1회만 돌리고 끝내지 마세요. 단, 같은 위반이 3회 연속 안 사라지면 멈추고 사람에게 보고 — 패턴 충돌이나 디자인 결정이 필요한 경우예요.

## 권한 건너뛰기 모드 (선택)

매번 "이 명령 실행해도 돼?" 묻는 게 귀찮으면 모드를 바꿀 수 있어요. 좌측 하단 모드 토글을 누르거나 `Shift + ⌘ + M` 단축키로 메뉴를 열고, **권한 건너뛰기** 를 선택하세요.

![Claude Code 모드 토글 — 권한 요청 / 편집 수락 / 계획 모드 / 자동 모드 / 권한 건너뛰기 중 권한 건너뛰기 선택됨](/img/install/skip-permission.png)

## 막힐 때

설치 단계 문제는 [Notion 가이드 · 막힐 때](https://www.notion.so/cashwalkteam/NudgeEAP-Design-System-MCP-35ea054b7d82807bb097c6c9d6b3d272) 를 먼저 확인해보세요.

### Claude 가 새 컴포넌트를 모름

⌘Q 로 완전 종료 후 재시작. 채팅창에서 `/mcp` 로 버전 확인.

### Claude 가 시각 자료 안 물어보고 바로 만들기 시작

> "잠깐, 시각 레퍼런스 먼저 확인해줘."

새 화면을 시작할 때마다 한 번은 묻는 게 정상이에요.

### 이전 화면 자료가 폴더에 남아 Claude 가 혼동

> "이거 새 화면이야. 이전 references.md 무시하고 새로 모아줘."

### 캐포비 화면인데 일반 어드민(antd) 으로 만듦

> "캐포비 어드민이야. NDS 의 캐포비 브랜드로 다시."

### 어드민이 마케팅 톤처럼 화려함

> "운영툴 톤으로 조밀하게. 강조 줄여줘."

### 검증을 한 번만 돌리고 끝냄 / 무한 자동 수정

한 번만 돌리고 끝내면:

> "한 번 더 돌려서 0건 확인하고 끝내줘."

같은 위반을 5번째 자동 수정 중이면:

> "그만. 같은 위반 몇 번째인지, 왜 안 풀리는지 알려줘."

### HTML 더블클릭했는데 빈 화면 / dev 서버도 비어 있음

> "라우팅 다시 잡아줘."
> "브라우저로 띄우고 콘솔 에러 확인해줘."

### 그 외

| 증상                       | 부탁 한 줄             |
| -------------------------- | ---------------------- |
| 시안 폴더를 못 찾는다고 함 | ⌘+O 로 시안 폴더 다시. |
| 영어로 답함                | "한국어로."            |

증상이 애매할 때는 만능 질문을 던져보세요.

> "지금 작업 폴더 / 커넥터 상태 점검해줘."

## 두 번째부터 — 빠른 워크플로

1. 시안 폴더 열고 30초 체크
2. (선택) 권한 건너뛰기 모드 ON
3. 시각 자료 1-2장을 시안 폴더에 넣기
4. "NDS 써서 [종류] 페이지 만들어줘 — PRD 첨부"
5. dev 서버 띄워 브라우저 확인
6. "NDS 써서 검증 2회 → HTML 한 파일로 → 사용 현황표까지" → `dist/index.html`
7. 슬랙·메일에 `dist/index.html` 첨부

한 페이지 시안은 보통 5분에서 30분이면 끝나요.
