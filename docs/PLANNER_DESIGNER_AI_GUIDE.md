# Planner & Designer AI Guide

이 문서는 기획자와 디자이너가 NudgeEAP 디자인 시스템 작업에서 AI를 어떻게 활용할 수 있는지 정리한 가이드입니다.

목적은 두 가지입니다.

- 디자인시스템을 더 빨리 이해하고 활용할 수 있게 돕기
- 반복적인 문서 정리, 비교, 초안 작성 작업을 AI로 보조하기

AI는 최종 판단자가 아니라 보조 도구로 사용합니다.  
특히 기획 의도, UX 우선순위, 최종 시각 결정은 사람이 확정합니다.

---

# 1. 언제 AI를 쓰면 좋은가

다음 같은 상황에서 도움이 됩니다.

## 기획자

- 새 화면을 설계하기 전에 기존 디자인시스템 컴포넌트로 가능한지 빠르게 점검할 때
- 요구사항 문서를 컴포넌트 기준으로 정리하고 싶을 때
- 상태 누락, 예외 케이스, 문구 길이 이슈를 미리 점검하고 싶을 때
- CTA가 많거나 정보가 과밀한 화면에서 우선순위와 배치를 점검하고 싶을 때
- 기존 화면을 디자인시스템 기준으로 바꾸려 할 때

## 디자이너

- 기존 서비스 화면에서 반복되는 UI 패턴을 빠르게 정리하고 싶을 때
- Figma 컴포넌트가 디자인시스템 컴포넌트와 어떻게 연결되는지 정리할 때
- 상태 매트릭스, Do / Don't, 사용/비사용 예시 초안을 만들 때
- Chip/Badge/안내문/CTA 강조가 과하지 않은지 빠르게 리뷰할 때
- 개발에 전달할 컴포넌트 설명이나 변경 요약을 작성할 때

---

# 2. AI로 잘 맡길 수 있는 작업

## 기획 문서 초안 정리

예:

- 어떤 컴포넌트를 써야 하는지 후보 정리
- 화면 요구사항을 상태별로 분해
- 엣지 케이스 체크리스트 만들기

## 디자인 패턴 비교

예:

- 기존 홈 화면과 웹뷰에서 같은 역할의 UI 찾기
- 검색창, 탭, 바텀시트처럼 공통화 가능한 패턴 찾기
- 현재 디자인시스템 컴포넌트로 대체 가능한지 검토하기
- CTA 그룹, 안내문 강조, 옵션 많은 드롭다운, 고밀도 리스트의 UX 기준 점검하기

## Figma 설명 보조

예:

- Figma 컴포넌트 설명 문구 초안 작성
- variant / size / state 설명 정리
- 개발 전달용 설명 문장 정리

## 문서 정리

예:

- Storybook과 Figma 간 연결 설명 초안
- 인벤토리용 usage summary 초안
- 릴리즈 노트 초안

# 3. AI에게 주면 좋은 입력

AI 결과 품질은 입력 정보에 크게 좌우됩니다.

가능하면 아래 정보를 함께 주는 것이 좋습니다.

- 화면 목적
- 사용자 행동
- 피그마 링크 또는 프레임 이름
- 참고하고 싶은 기존 화면
- 사용 후보 컴포넌트
- 필요한 상태
- 모바일/웹뷰 여부
- 문구 길이 제약

예시:

```text
회원가입 화면의 휴대폰 인증 영역을 정리하려고 해.
모바일 웹 기준이고, 입력창 + 인증 요청 버튼 + helper text가 필요해.
기존 홈페이지 화면과 디자인시스템 Input을 같이 참고해서
어떤 컴포넌트 조합으로 설계하면 좋을지 정리해줘.
필요한 상태와 예외 케이스도 함께 알려줘.
```

# 4. 추천 활용 시나리오

## 시나리오 A. 새 화면 기획 전 컴포넌트 후보 찾기

질문 예시:

```text
이 화면 요구사항으로 디자인시스템 컴포넌트 후보를 정리해줘.
이미 있는 컴포넌트로 가능한 것과 새로 필요할 수 있는 것을 나눠줘.
```

기대 결과:

- 기존 컴포넌트 사용 가능 항목
- 조합으로 해결 가능한 항목
- 신규 후보 항목

## 시나리오 B. Figma 컴포넌트 정의 문서 초안 만들기

질문 예시:

```text
이 Figma 컴포넌트를 기준으로
variant, size, state, 사용 예시, 사용하지 말아야 할 경우를 정리해줘.
디자인시스템 문서 초안처럼 써줘.
```

기대 결과:

- 문서 초안
- 상태 목록
- Do / Don't 예시

## 시나리오 C. 기획서에서 상태 누락 찾기

질문 예시:

```text
이 요구사항에서 빠질 수 있는 입력 상태나 예외 케이스를 찾아줘.
오류 상태, 빈 상태, 로딩 상태, disabled 상태를 우선 봐줘.
```

기대 결과:

- 상태 누락 목록
- 예외 케이스
- UX 확인 질문 목록

## 시나리오 D. 기존 화면을 디자인시스템 기준으로 옮기기 전 점검

질문 예시:

```text
이 기존 화면을 디자인시스템 기준으로 옮긴다면
어떤 부분은 기존 컴포넌트로 대응 가능하고
어떤 부분은 신규 컴포넌트나 recipe가 필요할지 정리해줘.
```

기대 결과:

- 바로 대체 가능한 부분
- 스타일만 조정하면 되는 부분
- 새 후보 컴포넌트 또는 recipe

# 5. 추천 프롬프트 템플릿

## 1. 기획자용 화면 요구사항 정리

```text
다음 화면 요구사항을 디자인시스템 기준으로 정리해줘.

- 화면 목적:
- 주요 사용자 행동:
- 필수 상태:
- 참고 화면:
- 모바일/웹:

아래 형식으로 답해줘.
1. 기존 컴포넌트로 가능한 부분
2. 조합으로 가능한 부분
3. 새로 검토할 컴포넌트 후보
4. 빠질 수 있는 상태나 예외 케이스
```

## 2. 디자이너용 Figma 컴포넌트 정리

```text
다음 Figma 컴포넌트를 디자인시스템 문서 초안처럼 정리해줘.

- 컴포넌트 이름:
- 목적:
- 상태:
- variant:
- size:
- 참고 화면:

아래 내용을 포함해줘.
- 언제 쓰는지
- 언제 쓰지 말아야 하는지
- 필요한 상태
- 개발 전달 시 주의할 점
```

## 3. 마이그레이션 검토용

```text
기존 구현과 디자인시스템 구현을 비교해서
차이가 날 수 있는 부분을 체크리스트로 정리해줘.

우선 확인 항목:
- spacing
- typography
- 버튼/인풋 상태
- empty/error 상태
- 모바일에서의 줄바꿈과 높이
```

## 4. 목업 UX 가드레일 점검용

```text
이 목업이 NudgeEAP DS UX 가드레일을 지키는지 점검해줘.

우선 확인 항목:
- primary solid 버튼이 한 화면의 대표 액션 1개에만 쓰였는지
- 우측 화살표 CTA가 반복되지 않는지
- Chip/Badge가 상태/분류/짧은 속성 표시가 아니라 장식으로 쓰이지 않았는지
- 안내문에 색 배경, 아이콘, 배지, 굵은 제목이 과하게 겹치지 않았는지
- 옵션 많은 드롭다운이나 정보 과밀 리스트의 배치가 적절한지

문제가 있으면 어떤 패턴으로 바꾸면 좋은지 함께 제안해줘.
```

# 6. AI를 쓸 때 주의할 점

AI가 잘하는 것

- 정리
- 비교
- 초안 작성
- 반복 패턴 발견
- 체크리스트 생성

AI가 단독으로 결정하면 안 되는 것

- 최종 UX 우선순위
- 실제 토큰 값 확정
- 브랜드 톤앤매너 확정
- Figma 컴포넌트 구조 최종 확정
- 접근성 충족 여부 최종 판단

권장 방식

- AI로 초안을 만든다
- 디자이너/기획자가 의도를 다시 확인한다
- 개발자가 구현 가능성과 현재 디자인시스템 범위를 확인한다

---

# 7. 앞으로 고려 중인 자동화 아이디어

현재는 AI를 “분석과 초안 작성” 중심으로 활용하고 있지만, 앞으로는 더 자동화할 수 있는 가능성도 있습니다.

예시 아이디어:

- Figma 링크와 Storybook 컴포넌트 메타데이터 자동 연결 보조
- 컴포넌트 인벤토리 초안 자동 생성
- 화면 요구사항 문서를 컴포넌트 체크리스트로 변환
- 기존 화면 스캔 후 공통 패턴 후보 추천
- 상태 매트릭스 초안 자동 생성
- 릴리즈 노트와 변경 영향 범위 초안 생성

이 자동화는 “AI가 대신 결정한다”는 의미가 아니라, 사람이 검토할 초안을 더 빠르게 만드는 방향으로 생각하고 있습니다.
