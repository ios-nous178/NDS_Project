# 컴포넌트 AI 컨텍스트 인덱스

이 문서는 AI 시스템이 Nudge Design System의 컴포넌트를 빠르게 파악하고 적절한 컴포넌트를 추천할 수 있도록 **전체 컴포넌트의 메타데이터와 사용 기준을 한 파일로 통합**한 인덱스입니다.

각 컴포넌트의 상세 Props, CSS 변수, 코드 예시는 개별 문서를 참조하세요.

## 사용 기준 요약

### Button

- **사용**: 폼 제출, 다이얼로그 확인/취소 등 사용자 액션을 트리거할 때
- **사용**: 상태를 변경하는 동작이 필요할 때 (저장, 삭제, 전송 등)
- **사용**: 워크플로우의 다음 단계로 진행할 때 (다음, 완료, 시작하기)
- **미사용**: 다른 페이지로 이동만 할 때
- **미사용**: 읽기 전용 상태 라벨을 표시할 때
- **미사용**: 선택/해제가 가능한 필터 태그일 때
- **주의**: primary solid는 화면의 대표 액션 1개만
- **주의**: ArrowNext/ChevronRight는 대표 전진 CTA 1개만, 반복 카드 CTA에는 사용하지 않음

### Badge

- **사용**: 상태(성공, 에러, 주의)나 카테고리를 읽기 전용 라벨로 표시할 때
- **사용**: 목록 항목에 짧은 메타 정보를 덧붙일 때 (신규, 완료, 대기 중 등)
- **사용**: 숫자 카운트를 강조할 때 (알림 뱃지 등)
- **미사용**: 사용자가 클릭하여 선택/해제하거나 삭제할 수 있는 태그
- **미사용**: 액션을 트리거하는 용도
- **미사용**: 긴 설명 텍스트를 담을 때

### Card

- **사용**: 관련 정보(이미지, 텍스트, 액션)를 하나의 시각적 단위로 묶어 표시할 때
- **사용**: 목록에서 각 항목을 카드 형태로 나열할 때 (상담 목록, 프로필 카드 등)
- **사용**: 콘텐츠 영역에 시각적 경계(그림자/테두리)를 줄 때
- **미사용**: 단순 구분선만 필요할 때
- **미사용**: 전체 페이지 레이아웃을 잡을 때
- **미사용**: 내부에 또 다른 Card를 중첩할 때

### Input

- **사용**: 사용자로부터 한 줄 텍스트를 입력받을 때 (이름, 이메일, 비밀번호 등)
- **사용**: 라벨, 에러 메시지, 도움말이 함께 필요한 폼 필드일 때
- **사용**: prefix/suffix 슬롯으로 단위나 아이콘을 함께 표시할 때
- **미사용**: 검색 전용 입력 (검색 아이콘 + 클리어 + Enter 트리거)
- **미사용**: 미리 정해진 옵션에서 선택
- **미사용**: 입력 필드 + 버튼을 한 줄로 조합
- **미사용**: 여러 줄 텍스트 입력

### SearchInput

- **사용**: 검색 기능을 제공할 때 (목록 필터링, 전체 검색 등)
- **사용**: 검색 아이콘 + 입력 + 클리어 버튼 + Enter 트리거 패턴이 필요할 때
- **미사용**: 일반 텍스트 입력 (이름, 이메일 등)
- **미사용**: 미리 정해진 옵션에서 필터링

### Select

- **사용**: 미리 정해진 옵션 목록에서 하나를 선택할 때
- **사용**: 옵션이 5개 이상이라 화면에 모두 노출하기 어려울 때
- **사용**: 폼 안에서 드롭다운 선택이 필요한 필드일 때
- **미사용**: 옵션이 3개 이하일 때 (Radio 사용)
- **미사용**: 복수 선택이 필요할 때 (Checkbox 또는 Chip 사용)
- **미사용**: 사용자가 직접 텍스트를 입력해야 할 때
- **주의**: 옵션 15개 초과는 검색 가능한 UI 검토, 50개 초과는 서버 검색/가상화 검토

### Checkbox & Radio

- **사용**: Checkbox - 여러 항목 중 복수 선택, 또는 단일 동의/약관 체크
- **사용**: Radio - 여러 항목 중 하나만 선택 (옵션이 2~5개일 때)
- **사용**: 모든 옵션을 한 번에 보여줘야 할 때
- **미사용**: 옵션이 많아 드롭다운이 적절할 때 (6개 이상)
- **미사용**: 선택 후 즉시 필터가 적용되는 인터랙티브 태그
- **미사용**: on/off 토글 (Switch 컴포넌트 사용)

### Chip

- **사용**: 사용자가 선택/해제/삭제할 수 있는 인터랙티브 태그일 때 (필터, 관심사, 태그 선택 등)
- **사용**: 선택된 항목을 시각적으로 표시하고 제거할 수 있게 할 때
- **사용**: 카테고리 필터를 토글 형태로 제공할 때
- **미사용**: 읽기 전용 상태 라벨
- **미사용**: 주요 액션 트리거
- **미사용**: 폼에서 명확한 복수 선택
- **미사용**: 안내문 강조, 섹션 제목 장식, 모든 카드에 반복되는 라벨
- **주의**: 라벨은 8자 안팎, 카드당 1~2개, 섹션당 2개 안팎

### Tab

- **사용**: 같은 맥락 내에서 뷰를 전환할 때 (프로필 탭, 설정 카테고리 등)
- **사용**: 콘텐츠를 논리적 그룹으로 나누되, 같은 페이지에서 전환해야 할 때
- **사용**: 탭 간 콘텐츠가 동등한 수준의 정보일 때
- **미사용**: 페이지 간 네비게이션
- **미사용**: 단계별 진행 (1단계 → 2단계 → 3단계)
- **미사용**: 콘텐츠 접기/펴기

### Modal

- **사용**: 복잡한 대화형 UI가 필요할 때 (폼 입력, 프로필 편집, 상세 정보 확인 등)
- **사용**: 사용자의 주의를 현재 작업에 강제로 집중시켜야 할 때
- **사용**: Header/Body/Footer 구조로 레이아웃을 나눠야 할 때
- **미사용**: 단순 확인/취소 알림
- **미사용**: 모바일에서 하단 슬라이드 UI
- **미사용**: 비차단 일시적 피드백
- **미사용**: 페이지 전체를 차지하는 콘텐츠

### Popup

- **사용**: 단순 확인/취소 판단만 필요할 때 (삭제 확인, 로그아웃 확인, 알림 메시지 등)
- **사용**: 제목 + 설명 + 확인/취소 버튼이면 충분한 간결한 대화상자일 때
- **사용**: Alert Dialog 패턴이 필요할 때 (alertdialog role)
- **미사용**: 폼 입력이나 복잡한 레이아웃이 필요한 대화상자
- **미사용**: 모바일에서 하단 목록 선택 UI
- **미사용**: 일시적 피드백 (저장 완료, 복사됨)

### BottomSheet

- **사용**: 모바일 화면에서 하단 슬라이드 UI가 필요할 때 (옵션 선택, 필터, 상세 정보 등)
- **사용**: 드래그로 닫을 수 있는 자연스러운 모바일 UX가 필요할 때
- **사용**: 화면 전체를 가리지 않으면서 추가 콘텐츠를 보여줄 때
- **미사용**: 데스크톱 전용 화면에서 대화상자가 필요할 때
- **미사용**: 단순 확인/취소 알림
- **미사용**: 간단한 드롭다운 선택

### Toast

- **사용**: 사용자 액션에 대한 일시적 피드백을 표시할 때 (저장 완료, 복사됨, 전송됨 등)
- **사용**: 사용자의 작업 흐름을 차단하지 않고 결과를 알릴 때
- **사용**: 되돌리기(Undo) 액션을 함께 제공할 때
- **미사용**: 사용자의 즉각적인 판단이 필요한 메시지
- **미사용**: 지속적으로 보여야 하는 정보
- **미사용**: 폼 유효성 에러

### ResultState

- **사용**: 데이터가 없는 빈 화면을 표시할 때 (검색 결과 없음, 목록이 비어있음 등)
- **사용**: 빈 상태에서 사용자에게 다음 행동을 안내할 때 (데이터 추가, 필터 변경 등)
- **사용**: 아이콘 + 제목 + 설명 + 액션 버튼 조합이 필요할 때
- **미사용**: 에러 페이지 (500, 404 등)
- **미사용**: 데이터 로딩 중
- **미사용**: 첫 사용 온보딩 가이드
