# Brand Chrome

브랜드 화면의 반복 chrome 을 DS 에서 바로 꺼내 쓰기 위한 컴포넌트 묶음입니다.
헤더, 푸터, 바텀네비의 구조와 아이콘 매핑은 DS 가 관리하고, 제품별 콘텐츠는 props 로 주입합니다.

## 사용 가능한 컴포넌트

| Brand    | Header                                | Footer           | BottomNav           |
| -------- | ------------------------------------- | ---------------- | ------------------- |
| Geniet   | `GenietAppBar`                        | `GenietFooter`   | `GenietBottomNav`   |
| Trost    | `TrostAppBar`                         | `TrostFooter`    | `TrostBottomNav`    |
| NudgeEAP | `NudgeEAPAppBar`, `NudgeEAPWebHeader` | `NudgeEAPFooter` | `NudgeEAPBottomNav` |
| Runmile  | `RunmileAppBar`, `RunmileWebHeader`   | `RunmileFooter`  | `RunmileBottomNav`  |
| CashwalkBiz | `CashwalkBizWebHeader`                   | `CashwalkBizFooter` | 웹 전용             |

## 호출 기준

- 앱/모바일 chrome 은 `{Brand}AppBar`, `{Brand}BottomNav` 를 우선 사용합니다.
- 웹 헤더가 별도로 있는 브랜드는 `{Brand}WebHeader` 를 사용합니다.
- 푸터는 `{Brand}Footer` 한 이름으로 호출하고, 필요한 경우 `surface="web"` 또는 `surface="app"` 으로 분기합니다.
- CashwalkBiz 는 웹 전용 브랜드라 AppBar / BottomNav 를 제공하지 않습니다.
- Geniet 은 앱 전용 푸터만 제공하므로 `GenietFooter` 는 app surface 만 사용합니다.

## HTML 목업

`@nudge-design/html`에서는 브랜드별 완성형 header/footer/bottom-nav wrapper 를 제공합니다. 로고 asset 은 기본적으로 `/brand-logos` 아래에서 찾고, 공유용 HTML처럼 경로가 다른 경우 `asset-base-url`만 바꿉니다.

```html
<nds-brand-header brand="nudge-eap" surface="web"></nds-brand-header>
<!-- ...page content... -->
<nds-brand-bottom-nav brand="nudge-eap" active-key="home"></nds-brand-bottom-nav>
<nds-brand-footer brand="nudge-eap"></nds-brand-footer>

<nds-trost-header></nds-trost-header>
<nds-trost-bottom-nav active-key="counsel"></nds-trost-bottom-nav>
<nds-trost-footer></nds-trost-footer>

<nds-brand-header
  brand="geniet"
  asset-base-url="https://cdn.example.com/nds/brand-logos"
></nds-brand-header>
```

`<nds-brand-bottom-nav brand="trost|geniet|nudge-eap|runmile" active-key="...">` 한 줄로 브랜드별 5탭(라벨/아이콘/색)이 자동 렌더됩니다. 제네릭 `nds-footer-tab-bar` 에 아이콘 SVG 를 직접 슬롯으로 주입하지 마세요. CashwalkBiz 는 웹 전용이라 bottom-nav 가 없습니다(빈 렌더).

브랜드 alias 는 `nds-nudge-eap-*`, `nds-trost-*`, `nds-geniet-*`, `nds-runmile-*`, `nds-cashwalk-biz-*` 형식을 사용합니다.

## 마이그레이션

```tsx
import {
  CashwalkBizFooter,
  CashwalkBizWebHeader,
  GenietAppBar,
  GenietBottomNav,
  GenietFooter,
  NudgeEAPBottomNav,
  NudgeEAPFooter,
  NudgeEAPWebHeader,
  RunmileBottomNav,
  RunmileFooter,
  TrostAppBar,
  TrostBottomNav,
  TrostFooter,
} from "@nudge-design/react";
```

기존 브랜드별 `AppFooter`, `WebFooter`, `DesktopFooter` 이름은 통합 푸터로 옮깁니다.

- `GenietAppFooter` → `GenietFooter`
- `TrostAppFooter` → `TrostFooter surface="app"`
- `TrostDesktopFooter` / `TrostWebFooter` → `TrostFooter surface="web"`
- `NudgeEAPAppFooter` → `NudgeEAPFooter surface="app"`
- `NudgeEAPWebFooter` → `NudgeEAPFooter surface="web"`
- `CashwalkBizWebFooter` → `CashwalkBizFooter`

## 관련 base 컴포넌트

- [`Header`](/components/app-bar): base `AppBar` / `WebHeader` 통합 API
- [`Footer`](/components/app-footer): base `AppFooter` / web footer 통합 API
