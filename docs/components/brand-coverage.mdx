# 브랜드 × 컴포넌트 커버리지

NDS 가 결국 갖춰야 할 **목표 컴포넌트** 리스트를 baseline 으로,
5개 브랜드 × 2개 패키지(`@nudge-design/react`, `@nudge-design/html`) 의 구현 현황을 보여줍니다.

범례 — `●` 코드 + 해당 브랜드 Figma 가이드 둘 다 ✓ · `○` 코드 있음, 이 브랜드 Figma 가이드 미정합 · `—` 코드 없음.

> 아래 보드는 Storybook `Coverage/Brand × Component` 와 **같은 공유 컴포넌트**
> (`@nudge-design/catalog` 의 `BrandCoverageTable`)로 렌더링됩니다. 판정·요약 계산은
> `scripts/coverage-logic.mjs` SSOT, 데이터는 `metadata/{tdsComponents,coverage-manifest}.json` 입니다.

<BrandCoverage />

## 갱신 방법

1. 브랜드별 Figma 가이드가 새로 추가되면 `metadata/tdsComponents.json` 의 해당 컴포넌트 `figmaByBrand` 객체에 URL 추가 → `○` 가 `●` 로 점등.
2. `brandChrome: true` 컴포넌트(예: Navbar→AppBar)는 `packages/react/src/{brand}/` 폴더에 파일이 있어야 코드 있음으로 판정됩니다.
3. `pnpm generate:brand-coverage` 실행 → `metadata/coverage-manifest.json`(보드 데이터 출처) 재생성.
