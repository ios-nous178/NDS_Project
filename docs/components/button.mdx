# Button

:::tip Figma 가이드 정합 완료 (2026-05-12)
Figma 컴포넌트 세트(node `508:6962`) 및 라이브러리 노드(`171:8385`) 기준으로 **사이즈, 패딩, 토큰, primary·secondary·neutral variant**가 정합 검증되었습니다.
이 페이지의 표/예제는 현재 `packages/react/src/Button.tsx`의 구현과 동기화된 값입니다.
:::

사용자의 클릭 액션을 처리하는 버튼 컴포넌트입니다. 4가지 변형(variant), 3가지 색상, 6가지 사이즈를 제공하며, 아이콘 슬롯을 통해 좌우에 아이콘을 배치할 수 있습니다.

```tsx
import { Button } from "@nudge-design/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- 폼 제출, 다이얼로그 확인/취소 등 **사용자 액션을 트리거**할 때
- 상태를 변경하는 동작이 필요할 때 (저장, 삭제, 전송 등)
- 워크플로우의 다음 단계로 진행할 때 (다음, 완료, 시작하기)

### 이럴 때 사용하지 마세요

- 다른 페이지로 이동만 할 때 → `<a>` 태그 또는 라우터의 Link를 사용하세요
- 읽기 전용 상태 라벨을 표시할 때 → [Badge](/components/badge)를 사용하세요
- 선택/해제가 가능한 필터 태그일 때 → [Chip](/components/chip)을 사용하세요

### 자주 하는 실수

- `onClick`에서 페이지 이동만 수행하는 Button — 스크린 리더가 링크로 인식하지 못합니다. 네비게이션에는 `<a>`를 사용하세요
- `disabled` 상태에서 비활성 이유를 안내하지 않음 — Tooltip이나 helperText로 이유를 함께 표시하세요
- `variant` 선택 기준 없이 사용 — 한 화면에서 `solid`는 1개(주요 CTA)만, 나머지는 `outlined` 이하로 위계를 나누세요
- 다른 페이지로 이동하는 CTA마다 우측 화살표 아이콘을 붙임 — ArrowNext/ChevronRight 계열은 대표 전진 CTA 1개에만 사용하세요
- 카드 리스트의 모든 "자세히 보기" 버튼에 화살표 반복 — 반복 CTA는 아이콘 없이 쓰거나 카드 전체 클릭 패턴을 검토하세요

## CTA 아이콘 정책

화살표 아이콘은 "이 화면에서 가장 중요한 다음 행동"을 빠르게 찾게 하는 장치입니다. 이동 버튼이라고 해서 항상 붙이지 않습니다.

- 한 뷰포트에서 화살표 CTA는 최대 1개를 권장합니다.
- 동일 섹션에 CTA가 여러 개라면 대표 CTA에만 화살표를 붙입니다.
- `outlined`, `soft` 같은 보조 버튼에는 기본적으로 화살표를 붙이지 않습니다.
- 반복 카드 안의 CTA에는 화살표를 반복하지 않습니다.

```tsx
// Good
<Button color="primary" variant="solid" rightIcon={<ArrowNextIcon />}>
  상담 신청하기
</Button>
<Button color="primary" variant="outlined">검사 시작하기</Button>
<Button color="neutral" variant="outlined">자세히 보기</Button>

// Avoid
<Button rightIcon={<ArrowNextIcon />}>상담 신청하기</Button>
<Button rightIcon={<ArrowNextIcon />}>검사 시작하기</Button>
<Button rightIcon={<ArrowNextIcon />}>자세히 보기</Button>
```

## Variant

4가지 스타일 변형을 제공합니다.

<ComponentPreview title="Variants">
  <Button variant="solid">Solid</Button>
  <Button variant="outlined">Outlined</Button>
  <Button variant="soft">Soft</Button>
  <Button variant="outlined">Outlined</Button>

```tsx
<Button variant="solid">Solid</Button>

```

| Variant        | 설명                         | 사용 시점                    |
| -------------- | ---------------------------- | ---------------------------- |
| `solid`        | 배경 채움 (기본값)           | 주요 CTA, 강한 강조          |
| `outlined`     | 테두리만 표시                | 보조 액션, solid와 함께 사용 |
| `soft`         | 연한 배경 + 컬러 텍스트      | 부드러운 강조                |

| Size    | 높이 | 폰트 / 라인 | 패딩 (좌우) | 아이콘 / Gap | 사용 시점                |
| ------- | ---- | ----------- | ----------- | ------------ | ------------------------ |
| `xl`    | 52px | 16 / 24px   | 16px        | 20 / 8px     | 하단 CTA 등 큰 버튼      |
| `lg`    | 48px | 16 / 24px   | 16px        | 20 / 8px     | 기본 버튼 (기본값)       |
| `md`    | 44px | 15 / 22px   | 24px        | 20 / 8px     | 카드 내부 등 중간 크기   |
| `sm`    | 42px | 14 / 20px   | 16px        | 20 / 8px     | 리스트 액션 등 작은 버튼 |
| `xs`    | 38px | 13 / 18px   | 16px        | 18 / 6px     | 인라인, 태그 옆 등       |
| `field` | 48px | 15 / 22px   | 16px        | 20 / 8px     | Input 옆 연동 버튼       |

## 아이콘

`leftIcon`, `rightIcon` prop으로 아이콘을 배치합니다.

```tsx
import { SearchIcon, ChevronRightIcon } from "@nudge-design/icons";

<Button leftIcon={<SearchIcon />}>검색</Button>
<Button rightIcon={<ChevronRightIcon />}>다음</Button>
<Button leftIcon={<SearchIcon />} rightIcon={<ChevronRightIcon />}>검색</Button>
```

## SlotProps

```tsx
interface ButtonSlotProps {
  label?: React.HTMLAttributes<HTMLSpanElement>;
  leftIcon?: React.HTMLAttributes<HTMLSpanElement>;
  rightIcon?: React.HTMLAttributes<HTMLSpanElement>;
}
```

```tsx
<Button
  slotProps={{
    label: { className: "tracking-wide" },
    leftIcon: { style: { marginRight: 4 } },
  }}

  슬롯 커스텀
```

## data-slot 구조

```
button [data-slot="root"] [data-variant] [data-size] [data-color]
├── span [data-slot="left-icon"]    (leftIcon이 있을 때)
├── span [data-slot="label"]
└── span [data-slot="right-icon"]   (rightIcon이 있을 때)
```

styled-components에서 타겟팅:

```tsx
const StyledButton = styled(Button)`
  [data-slot="label"] {
    letter-spacing: 0.05em;
  }
`;
```

## 접근성

- `type="button"` 기본값으로 form 의도치 않은 submit 방지
- `disabled` 시 `cursor: default` 및 색상 변경
- `ref` 전달 지원 (`React.forwardRef`)
