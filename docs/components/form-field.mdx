# FormField

라벨 · 설명 · 헬퍼 · 에러 · 글자수 카운터를 한 곳에서 다루는 입력 필드 래퍼입니다. 내부 컨트롤(Input, Select, Textarea, RadioGroup 등)은 `children`으로 받고, 라벨 · 헬퍼 · 에러 · 카운터 영역을 일관된 톤/간격으로 제공합니다. 단일 컨트롤에 라벨과 메시지를 일관되게 붙일 때 사용하세요.

```tsx
import { FormField } from "@nudge-design/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- `Input`, `Select`, `Textarea` 등 **단일 컨트롤에 라벨**과 헬퍼/에러를 일관되게 붙일 때
- 필수 표시(`*`) 또는 선택 표시(`(선택)`)를 통일된 스타일로 노출해야 할 때
- 글자 수 제한이 있는 텍스트 영역에 **카운터**를 함께 표시하고 싶을 때

### 이럴 때 사용하지 마세요

- 라벨/헬퍼가 필요 없는 검색바·인라인 입력 → `Input` / `SearchInput`을 직접 사용하세요
- 라디오/체크박스 묶음의 그룹 라벨이 자체적으로 필요 → 묶음 컴포넌트의 라벨 prop을 사용하세요

---

## 기본 사용

```tsx
<FormField label="이메일" required helper="로그인 시 사용됩니다">
  <Input placeholder="you@example.com" />
</FormField>
```

### 설명 · 에러 · 카운터

```tsx
<FormField
  label="자기소개"
  description="다른 회원에게 보여지는 한 줄 소개예요"
  error={text.length > 50 ? "50자 이내로 작성해주세요" : undefined}
  counter={`${text.length} / 50`}
>
  <Textarea value={text} onChange={(e) => setText(e.target.value)} />
</FormField>
```

`error`가 존재하면 helper 대신 에러 텍스트가 노출됩니다. counter는 항상 우측에 함께 표시됩니다.

### 선택 입력 표시

```tsx
<FormField label="회사명" optional>
  <Input />
</FormField>
```

`required={true}`가 우선합니다 (둘 다 켜져 있으면 `*`만 표시).

### 외부 컨트롤 id 연결

라벨 클릭으로 외부 컨트롤에 포커스를 주고 싶다면 `htmlFor`를 직접 지정합니다.

```tsx
<FormField label="비밀번호" htmlFor="pw">
  <input id="pw" type="password" />
</FormField>
```

## data-slot 구조

```
div [data-slot="root"]
├── div [data-slot="label-row"]                  (label 또는 description이 있을 때)
│   ├── label [data-slot="label"] (htmlFor)
│   │   ├── span (라벨 텍스트)
│   │   ├── span.nds-form-field__required   ("*", required=true)
│   │   └── span.nds-form-field__optional   ("(선택)", optional=true && !required)
│   └── div [data-slot="description"]
├── div [data-slot="control"]                    (children 래퍼, display: contents)
│   └── { children }
└── div [data-slot="footer"]                     (helper / error / counter 중 1+개일 때)
    ├── span [data-slot="error"] (role="alert")  또는
    ├── span [data-slot="helper"]                또는
    └── span [data-slot="counter"]
```

`label`의 `htmlFor`는 `htmlFor` prop이 있으면 그 값, 없으면 `useId()`로 자동 생성된 id가 사용됩니다. 자동 생성 id의 경우 children의 컨트롤이 동일 id를 받는 책임은 호출자에게 있습니다.

## CSS 변수

이 컴포넌트는 CSS 변수를 노출하지 않습니다. 라벨/메시지 타이포·색상은 시멘틱 토큰(`cv.text.default`, `cv.text.subtle`, `cv.text.disabled`, `cv.error.main`, `typeScale.body3`, `typeScale.caption1`)에 직접 바인딩됩니다.

---

## 접근성

- 라벨은 네이티브 `<label htmlFor>`로 컨트롤과 연결
- `error`가 있을 때 메시지는 `role="alert"`로 변경 사항을 즉시 안내
- helper / error 메시지에는 `useId()` 기반의 `id`가 부여됨 (`{fieldId}-helper`, `{fieldId}-error`) — 컨트롤에서 `aria-describedby`로 연결하면 스크린 리더에 메시지가 함께 읽힘
- `*` 필수 표시와 `(선택)` 텍스트는 `aria-hidden="true"`라서 스크린 리더는 라벨 텍스트만 읽음 — 의미가 필요한 경우 컨트롤에 `aria-required`를 직접 부여하세요
