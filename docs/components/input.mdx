# Input

:::tip Figma 가이드 정합 완료 (2026-05-12)
Figma 컴포넌트(node `430:4212`) 및 라이브러리 노드(`171:9903`, Section_Input `294:12`) 기준으로 **사이즈, gap, helpertext variant 컬러(default/success/error/disabled), 아이콘 사이즈**가 정합 검증되었습니다.
이 페이지의 표/예제는 현재 `packages/react/src/Input.tsx`의 구현과 동기화된 값입니다.
:::

텍스트 입력 필드 컴포넌트입니다. 라벨, 도움 텍스트, 에러 상태, 클리어 버튼, prefix/suffix 슬롯을 지원합니다. Flat API와 Compound API를 모두 제공합니다.

```tsx
import { Input } from "@nudge-eap/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- 사용자로부터 **한 줄 텍스트**를 입력받을 때 (이름, 이메일, 비밀번호 등)
- 라벨, 에러 메시지, 도움말이 함께 필요한 폼 필드일 때
- prefix/suffix 슬롯으로 단위나 아이콘을 함께 표시할 때

### 이럴 때 사용하지 마세요

- 검색 전용 입력 (검색 아이콘 + 클리어 + Enter 트리거) → [SearchInput](/docs/components/search-input)을 사용하세요
- 미리 정해진 옵션에서 선택 → [Select](/docs/components/select)를 사용하세요
- 입력 필드 + 버튼을 한 줄로 조합 → [FieldActionRow](/docs/components/field-action-row)를 사용하세요
- 여러 줄 텍스트 입력 → `<textarea>`를 사용하세요

### 자주 하는 실수

- `label` 없이 `placeholder`만 사용 — placeholder는 입력 시 사라지므로 접근성과 UX 모두 나빠집니다. 항상 `label`을 제공하세요
- 에러 상태에서 `errorMessage` 미제공 — `error` prop만으로는 사용자가 무엇이 잘못인지 알 수 없습니다

### 클리어 버튼

```tsx
const [value, setValue] = useState("");

<Input
  label="검색"
  value={value}
  onChange={(e) => setValue(e.target.value)}
  clearable
  onClear={() => setValue("")}
  placeholder="검색어를 입력하세요"
/>;
```

### Prefix / Suffix

```tsx
import { SearchIcon } from "@nudge-eap/icons";

  prefix={<SearchIcon />}

  suffix={<Button size="xs" variant="soft">전송</Button>}
  placeholder="메시지 입력"
/>
```

## Compound API

각 요소를 직접 조합하여 자유로운 레이아웃을 구성합니다.

```tsx
<Input.Root error>
  <Input.Label>이메일</Input.Label>
  <Input.Wrapper>
    <Input.Field placeholder="example@email.com" />
    <Input.ClearButton onClick={() => setValue("")} />
  </Input.Wrapper>
  <Input.Helper error>올바른 이메일 형식이 아닙니다</Input.Helper>
</Input.Root>
```

### 커스텀 레이아웃

```tsx
<Input.Root>
  <div style={{ display: "flex", justifyContent: "space-between" }}>
    <Input.Label>비밀번호</Input.Label>
    <a href="/forgot">비밀번호 찾기</a>
  </div>
  <Input.Wrapper>
    <Input.Field type="password" placeholder="8자 이상" />
  </Input.Wrapper>
  <Input.Helper>영문, 숫자, 특수문자를 포함해주세요</Input.Helper>
</Input.Root>
```

## Flat API Props

| Prop               | 타입             | 기본값  | 설명                               |
| ------------------ | ---------------- | ------- | ---------------------------------- |
| `label`            | `string`         | —       | 라벨 텍스트                        |
| `helperText`       | `string`         | —       | 도움 텍스트                        |
| `error`            | `boolean`        | `false` | 에러 상태                          |
| `errorMessage`     | `string`         | —       | 에러 메시지 (helperText 대신 표시) |
| `clearable`        | `boolean`        | `false` | 클리어 버튼 표시                   |
| `onClear`          | `() => void`     | —       | 클리어 콜백                        |
| `prefix`           | `ReactNode`      | —       | 좌측 addon (아이콘 등)             |
| `suffix`           | `ReactNode`      | —       | 우측 addon (버튼 등)               |
| `fullWidth`        | `boolean`        | `true`  | 부모 너비 100% 채움                |
| `disabled`         | `boolean`        | `false` | 비활성화                           |
| `readOnly`         | `boolean`        | `false` | 읽기 전용                          |
| `slotProps`        | `InputSlotProps` | —       | 내부 슬롯별 props 전달             |
| `wrapperClassName` | `string`         | —       | Wrapper className                  |
| `wrapperStyle`     | `CSSProperties`  | —       | Wrapper 인라인 스타일              |

## Compound 컴포넌트

### Input.Root

| Prop        | 타입      | 기본값  | 설명      |
| ----------- | --------- | ------- | --------- |
| `error`     | `boolean` | `false` | 에러 상태 |
| `disabled`  | `boolean` | `false` | 비활성화  |
| `readOnly`  | `boolean` | `false` | 읽기 전용 |
| `fullWidth` | `boolean` | `true`  | 전체 너비 |

### Input.Label

`<label>` 요소입니다. `htmlFor`가 Input.Field의 `id`와 자동으로 연결됩니다.

### Input.Wrapper

입력 필드를 감싸는 컨테이너입니다. focus, error, disabled, readonly 상태에 따라 border 색상이 변경됩니다.

### Input.Field

실제 `<input>` 요소입니다. `ref` 전달을 지원합니다.

### Input.ClearButton

입력값 삭제 버튼입니다. `aria-label="입력 삭제"`가 내장되어 있습니다.

### Input.Helper

도움 텍스트/에러 메시지를 표시합니다. Section_Input(`294:12`) 기준 4가지 variant 컬러를 지원합니다.

| Prop      | 타입                                              | 기본값      | 설명                                          |
| --------- | ------------------------------------------------- | ----------- | --------------------------------------------- |
| `error`   | `boolean`                                         | —           | 에러 스타일 + `role="alert"` (variant="error"와 동일) |
| `variant` | `"default" \| "success" \| "error" \| "disabled"` | `"default"` | 헬퍼 텍스트 색상 변형                         |
| `icon`    | `ReactNode`                                       | —           | 텍스트 앞 16×16 아이콘 (color는 헬퍼 컬러 상속) |

#### Helper variant 컬러 (Figma Section_Input 294:12 기준)

| variant    | 토큰                                | 컬러      | 용도                       |
| ---------- | ----------------------------------- | --------- | -------------------------- |
| `default`  | Text/Muted/Default                  | `#999999` | 일반 도움말                |
| `success`  | Text/Brand/Default                  | `#2B96ED` | 폼 검증 통과 (브랜드 톤)   |
| `error`    | Text/Status/Error                   | `#F13F00` | 폼 오류                    |
| `disabled` | Text/Disabled/Default               | `#C7C7C7` | 비활성 상태                |

> Banner/Toast의 success(녹색)와 의도적으로 구분 — 폼 검증 통과는 브랜드 톤을 사용합니다.

## data-slot 구조

```
div [data-slot="root"]
├── label [data-slot="label"]
├── div [data-slot="wrapper"]  [data-focused] [data-error] [data-disabled] [data-readonly]
│   ├── (prefix)
│   ├── input [data-slot="field"]
│   ├── button [data-slot="clear"]   (clearable일 때)
│   └── (suffix)
└── span [data-slot="helper"]  [data-error]
```

## 접근성

- `<label>`과 `<input>`이 `htmlFor`/`id`로 자동 연결
- 클리어 버튼에 `aria-label="입력 삭제"` 내장
- `ref` 전달 지원
