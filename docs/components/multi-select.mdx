# MultiSelect

검색 + 전체선택/해제 + 체크박스 리스트 + 취소/적용 푸터를 가진 다중 선택 필터 드롭다운입니다. 패널 안에서 초안(draft)을 편집하고 **적용**을 눌러야 `onValueChange`가 발화합니다.

```tsx
import { MultiSelect } from "@nudge-design/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- 리포트 상단의 "광고 다중 선택" 처럼 **여러 항목을 골라 거르는 필터**가 필요할 때
- 선택을 즉시 반영하지 않고, 패널에서 묶어 고른 뒤 **적용 시점에만 한 번 반영**하고 싶을 때
- 옵션이 많아 **검색**으로 좁히고, **전체선택/해제**로 빠르게 토글하고 싶을 때
- `SearchInput` / `Select` / `DateRangePicker` 와 함께 한 줄 필터 바를 가로로 조합할 때

### 이럴 때 사용하지 마세요

- 단일 선택이고 즉시 반영되어야 할 때 → Select를 사용하세요 (적용 버튼이 불필요한 마찰입니다)
- 계층(시/도 ▸ 시/군/구) 다중 선택 → CheckboxTree를 사용하세요
- 항상 펼쳐진 인라인 체크 리스트(약관 동의 등) → CheckboxGroup을 사용하세요
- 카테고리 칩 토글로 거르는 필터 → FilterBar를 사용하세요

### 자주 하는 실수

- **단일 선택에 MultiSelect를 쓰는 것** — MultiSelect는 적용 버튼이 있는 다중 필터 전용입니다. 즉시 반영 단일 드롭다운에는 과합니다.
- **적용 전 `onValueChange`를 기대하는 것** — 적용(apply) 전까지 발화하지 않습니다. 취소/바깥클릭은 초안을 폐기합니다. `value`(적용값)와 패널 내 draft를 혼동하지 마세요.
- **빈 상태를 직접 그리는 것** — 검색 결과 0건이면 자동으로 빈 상태("검색 결과가 없습니다.")가 노출됩니다.
- **전체선택/해제를 전체 목록 기준으로 가정하는 것** — 전체선택/해제는 **현재 검색 필터된 항목** 기준으로 토글됩니다(비활성 항목 제외).

## 기본 사용법

```tsx
const [selectedAds, setSelectedAds] = useState<string[]>([]);

<MultiSelect
  options={[
    { value: "a", label: "캠페인 A 타겟팅" },
    { value: "b", label: "캠페인 B 리타겟" },
    { value: "c", label: "캠페인 C 브랜딩", disabled: true },
  ]}
  value={selectedAds}
  onValueChange={setSelectedAds}
  placeholder="모든 광고"
  searchPlaceholder="광고명으로 검색"
/>
```

`value`는 적용된(commit) 선택 값이고, `onValueChange`는 **적용 버튼 클릭 시에만** 호출됩니다. 패널을 열 때마다 초안은 현재 `value`로 리셋되며, 취소·바깥클릭·스크롤·Esc는 초안을 폐기하고 닫습니다.

트리거 요약은 선택이 없으면 `placeholder`, 있으면 기본 포맷 `${n}개 선택`으로 표시됩니다. 포맷을 바꾸려면 `formatSummary`를 넘기세요.

```tsx
<MultiSelect
  options={ads}
  value={selectedAds}
  onValueChange={setSelectedAds}
  formatSummary={(count, total) => `${count}/${total} 광고`}
/>
```

검색창을 숨기려면 `searchable={false}` 를 사용합니다.

## CSS 변수

이 컴포넌트는 자체 `--nds-multi-select-*` CSS 변수를 노출하지 않습니다. 트리거 높이와 여백은 다른 입력 계열과 공유하는 토큰을 참조합니다 — `--nds-select-height`(Select와 동일 높이), `--semantic-inset-input`, `--semantic-gap-tight`, `--semantic-gap-comfortable`. 색·라운드·그림자는 토큰에서 자동 결정되며 raw hex를 직접 지정하지 마세요.

## data-slot 구조

```
div [data-slot="root"] [data-open] [data-disabled] [data-fullwidth]
├── button [data-slot="trigger"] [data-open] [data-has-value] [data-error] [data-disabled]
│   │   (aria-haspopup="listbox" / aria-expanded / aria-invalid)
│   ├── span [data-placeholder]            (요약 텍스트 / placeholder)
│   └── span [data-slot 없음, chevron]     (aria-hidden, 펼침 시 회전)
└── div [data-slot="dropdown"]             (role="listbox" / aria-multiselectable, open 일 때만)
    ├── div [data-slot="search"]           (searchable 일 때 — 돋보기 + input[autoFocus])
    ├── button [data-slot="select-all"]    (전체선택/해제 + 선택 개수)
    │   └── span [data-checked]            (체크 인디케이터, aria-hidden)
    ├── div [data-slot="list"]
    │   ├── div [data-slot="empty"]        (필터 결과 0건일 때만)
    │   └── label [role="option"] [aria-selected] [data-checked] [data-disabled]  (옵션마다)
    │       ├── input[type=checkbox]       (시각적으로 숨김 — 인디케이터로 대체)
    │       ├── span [data-checked]        (체크 인디케이터, aria-hidden)
    │       └── span [data-slot 없음, label]
    └── div [data-slot="footer"]
        ├── button [data-variant="cancel"] (취소 — 초안 폐기 후 닫기)
        └── button [data-variant="apply"]  (적용 — onValueChange 발화 후 닫기)
```

## 접근성

- 트리거는 `aria-haspopup="listbox"`, 펼침 상태는 `aria-expanded`로 노출합니다. `error`일 때 `aria-invalid`가 부여됩니다.
- 드롭다운 패널은 `role="listbox"` + `aria-multiselectable="true"` 이며, 각 옵션은 `role="option"` + `aria-selected`(초안 체크 상태)로 표시됩니다.
- 패널이 열리면 검색 input에 자동 포커스됩니다(`searchable`일 때). 옵션의 실제 체크박스 input은 시각적으로 숨겨져 있고 화면 표시는 인디케이터가 담당합니다.
- Esc 키로 패널을 닫습니다(초안 폐기). 바깥 클릭·스크롤도 dismissable 레이어로 동일하게 취소 후 닫힙니다.
- chevron, 체크 인디케이터, 검색 아이콘 등 장식 요소는 `aria-hidden`으로 보조기기에서 제외됩니다.
