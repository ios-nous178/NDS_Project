# 컴포넌트

NudgeEAP Design System이 제공하는 15개 React 컴포넌트입니다. **어떤 컴포넌트를 써야 할지 모르겠다면** 바로 아래 선택 가이드부터 확인하세요.

각 컴포넌트 문서에는 "사용 가이드" 섹션이 있어 **언제 쓰는지, 언제 쓰지 않는지, 자주 하는 실수**를 확인할 수 있습니다.

---

## 컴포넌트 선택 가이드

어떤 컴포넌트를 써야 할지 모르겠다면, 아래 의사결정 트리를 따라가세요.

### 사용자에게 확인을 받아야 할 때

```
확인/취소가 필요한가?
├── 단순 확인/취소 (텍스트 + 버튼) → Popup
├── 폼 입력이나 복잡한 UI → Modal
└── 모바일에서 하단 슬라이드 UI → BottomSheet
```

### 사용자에게 피드백을 줄 때

```
사용자 흐름을 차단해야 하는가?
├── 예 → Popup (확인을 눌러야 닫힘)
└── 아니오
    ├── 일시적 메시지 (3초 후 사라짐) → Toast
    └── 데이터가 없는 빈 화면 → EmptyState
```

### 사용자 입력을 받을 때

```
어떤 종류의 입력인가?
├── 자유 텍스트 한 줄 → Input
├── 검색 → SearchInput
├── 입력 + 버튼 조합 (인증번호 등) → FieldActionRow
├── 미리 정해진 옵션에서 선택
│   ├── 옵션 5개 이상 → Select
│   ├── 옵션 2~5개, 하나만 선택 → Radio
│   └── 옵션 2~5개, 복수 선택 → Checkbox
└── 필터 태그 토글 → Chip
```

### 정보를 표시할 때

```
어떤 종류의 표시인가?
├── 상태/카테고리 읽기 전용 라벨 → Badge
├── 관련 정보를 카드로 묶기 → Card
└── 선택/해제 가능한 인터랙티브 태그 → Chip
```

---

### 유사 컴포넌트 비교

#### 오버레이 3종: Modal vs Popup vs BottomSheet

|  | Modal | Popup | BottomSheet |
|---|---|---|---|
| 용도 | 복잡한 대화형 UI | 단순 확인/취소 | 모바일 하단 슬라이드 |
| ARIA role | `dialog` | `alertdialog` | `dialog` |
| 내부 구조 | Header / Body / Footer | TextInfo + Actions | Handle / Header / Body / Footer |
| 폼 입력 가능 | O | X | O |
| 드래그 닫기 | X | X | O (핸들) |
| 주 사용 환경 | 데스크톱 + 모바일 | 공통 | 모바일 우선 |

#### Badge vs Chip

|  | Badge | Chip |
|---|---|---|
| 인터랙션 | 읽기 전용 | 선택/해제/삭제 가능 |
| 용도 | 상태 라벨 (성공, 에러) | 필터 태그, 관심사 선택 |
| 클릭 가능 | X | O |

#### Input vs SearchInput vs FieldActionRow

|  | Input | SearchInput | FieldActionRow |
|---|---|---|---|
| 용도 | 범용 텍스트 입력 | 검색 전용 | 입력 + 버튼 조합 |
| 검색 아이콘 | X | O (내장) | X |
| 버튼 포함 | X | X (Enter 트리거) | O |
| 라벨/에러 | O | 제한적 | O (Input 위임) |

## 공통 특징

### Hybrid API

모든 컴포넌트는 두 가지 사용 방식을 제공합니다.

```tsx
// Flat — 간단하게
<Input label="이름" placeholder="입력하세요" error errorMessage="필수 항목입니다" />

// Compound — 자유롭게
<Input.Root error>
  <Input.Label>이름</Input.Label>
  <Input.Wrapper>
    <Input.Field placeholder="입력하세요" />
    <Input.ClearButton onClick={handleClear} />
  </Input.Wrapper>
  <Input.Helper error>필수 항목입니다</Input.Helper>
</Input.Root>
```

### CSS 변수 기반 스타일링

모든 컴포넌트의 시각적 속성은 CSS 변수로 노출됩니다. 프로젝트 테마에 맞게 오버라이드할 수 있습니다.

```css
/* 프로젝트 전역 오버라이드 */
:root {
  --nds-button-radius: 12px;
  --nds-input-radius: 12px;
}
```

### data-slot 속성

각 내부 요소는 `data-slot` 속성을 가지며, 이를 통해 styled-components나 CSS에서 타겟팅할 수 있습니다.

```css
[data-slot="label"] {
  font-weight: 600;
}
```

### 접근성

- Modal/Popup: 포커스 트랩, ESC 닫기, `aria-modal`, `aria-labelledby` 내장
- Input: `label`과 `htmlFor` 자동 연결
- Button: `type="button"` 기본값, `disabled` 상태 처리
