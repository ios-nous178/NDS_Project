# SelectionButtonGroup

폼 안에서 상호 배타적인 옵션을 단일 선택하는 버튼 그룹입니다. 브랜드색 아웃라인의 개별 버튼을 gap으로 나열하며, `FormField`의 ContentSlot에 교체해 사용합니다.

```tsx
import { SelectionButtonGroup } from "@nudge-design/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- 폼 내에서 **상호 배타적인 옵션 2~3개**를 단일 선택할 때 (성별, 노출 구분, "전체 / 특정 시간만" 등)
- 설명이나 아이콘 없이 **짧은 라벨**만으로 구분되는 선택지일 때
- `FormField`의 입력 슬롯을 라디오 그룹 대신 브랜드색 버튼 그룹으로 교체하고 싶을 때
- 라벨 길이가 달라도 옵션들의 너비를 **등폭**으로 자동 정렬하고 싶을 때

### 이럴 때 사용하지 마세요

- 옵션이 4개 이상 → 가로 폭이 부족합니다. Select 또는 SelectionCard를 사용하세요.
- 라벨 + 설명 + 아이콘이 필요한 카드형 선택 → SelectionCard가 적합합니다.
- 뷰/상태 전환(목록·그리드 토글 등) → 폼 입력 컴포넌트라 위계가 어색합니다. variant="segment"의 Tab을 사용하세요.

### 자주 하는 실수

- **Tab variant="segment"과 혼동** — segment는 연결된 회색 트랙(뷰/상태 전환)이고, SelectionButtonGroup은 폼 입력(개별 브랜드색 버튼)입니다. 폼 안 단일 선택이면 이 컴포넌트입니다.
- **선택색을 hex로 박기** — selected 상태는 `--semantic-bg-brand-subtle` / `--semantic-border-brand-default` 캐스케이드로 5개 브랜드에 자동 대응합니다. 직접 색을 지정하지 마세요.
- **손으로 width 지정** — 그룹 내 옵션은 등폭이 기본입니다. "전체"(좁음) / "특정 지역"(넓음)처럼 라벨 길이가 달라도 컴포넌트가 가장 넓은 옵션 기준으로 자동 균등하게 그립니다. 컨테이너 100%로 늘리려면 `fullWidth`를 쓰세요.
- **"특정 X" 선택 결과를 또 다른 노란 outlined 버튼·칩으로 인라인 나열** — 선택 버튼과 시각적으로 동일해 혼동됩니다. 소수 고정 선택지는 toggle Chip, 동적 다중 선택은 SelectedItemsPanel + SelectedItemRow로 그리세요.

## 기본 사용법

```tsx
const [value, setValue] = useState("always");

<SelectionButtonGroup
  value={value}
  onValueChange={setValue}
  options={[
    { value: "always", label: "항상" },
    { value: "time", label: "특정 시간만" },
    { value: "weekday", label: "특정 요일/시간만" },
  ]}
/>
```

그룹 내 옵션은 라벨 길이와 무관하게 가장 넓은 옵션 기준으로 **등폭** 정렬됩니다.

### 컨테이너 100% 분할 (fullWidth)

기본값에서 그룹은 콘텐츠 너비에 hug됩니다. 컨테이너 전체 너비를 옵션 수만큼 균등 분할하려면 `fullWidth`를 켜세요.

```tsx
<SelectionButtonGroup
  fullWidth
  value={value}
  onValueChange={setValue}
  options={[
    { value: "all", label: "전체" },
    { value: "specific", label: "특정 성별" },
  ]}
/>
```

### 비활성화

그룹 전체를 끄려면 `disabled`를, 개별 옵션만 끄려면 옵션의 `disabled`를 사용합니다.

```tsx
<SelectionButtonGroup
  value={value}
  onValueChange={setValue}
  options={[
    { value: "all", label: "전체" },
    { value: "male", label: "남성" },
    { value: "female", label: "여성", disabled: true },
  ]}
/>
```

## data-slot 구조

```
div [data-slot="root"] [data-fullwidth="true|false"]   (role="radiogroup")
└── button [data-slot="item"] [data-selected="true|false"]   (role="radio", aria-checked, type="button")
    └── span (label — 옵션의 label, 좁아지면 ellipsis 말줄임)
```

---

## 접근성

- 루트는 `role="radiogroup"`, 각 옵션 버튼은 `role="radio"` + `aria-checked`로 단일 선택 시맨틱을 표현합니다.
- 각 버튼은 `type="button"`이라 폼 안에서 의도치 않은 submit을 일으키지 않습니다.
- 이미 선택된 값이나 비활성 옵션을 클릭해도 `onValueChange`가 호출되지 않습니다.
- 키보드 포커스 시 `:focus-visible`로 브랜드색 outline(2px, offset 2px)을 표시합니다.
- 비활성 옵션은 네이티브 `disabled` 속성으로 처리되어 포커스/클릭이 차단됩니다.
