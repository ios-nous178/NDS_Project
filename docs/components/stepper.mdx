# Stepper

다단계 흐름의 진행 단계 인디케이터입니다. 시각화 전용으로, 실제 폼 상태 관리는 `MultiStepForm`을 사용합니다.

```tsx
import { Stepper } from "@nudge-design/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- 회원가입, 결제, 검사 진행 등 **순차적인 단계**의 위치를 보여줄 때
- 완료된 단계 / 현재 단계 / 남은 단계를 한눈에 보여줘야 할 때

### 이럴 때 사용하지 마세요

- 값 증감(수량 조절 등) → NumberStepper를 사용하세요
- 단계별 폼 상태 관리·검증 → MultiStepForm을 사용하세요
- 같은 페이지 내 뷰 전환 → [Tabs](/components/tabs)를 사용하세요

> **Stepper vs MultiStepForm** — Stepper는 진척을 **표시만** 합니다(상태는 `current` 인덱스로 자동 계산, per-step status 없음). 폼 콘텐츠 + 단계 네비게이션 + 검증을 한 컨테이너로 묶으려면 [MultiStepForm](/components/multi-step-form)을 쓰세요. 시간순 이벤트 로그는 [Timeline](/components/timeline).

## 기본 사용

```tsx
<Stepper
  current={1}
  steps={[
    { key: "terms", label: "약관 동의" },
    { key: "info", label: "정보 입력" },
    { key: "done", label: "완료" },
  ]}
/>
```

`current`(0-based) 미만은 `completed`(체크 아이콘), 같으면 `current`, 큰 인덱스는 `upcoming`으로 자동 표시됩니다.

### Dots 변형

라벨 없이 점만 표시할 때 사용합니다.

```tsx
<Stepper variant="dots" current={2} steps={[
  { key: "1" }, { key: "2" }, { key: "3" }, { key: "4" }
]} />
```

### Bar 변형

가로 막대 + 2단 라벨(스텝 번호 + 제목)로, 캐시워크 for Business 어드민의 다단계 흐름에 사용합니다(구 StepProgress 흡수). `label`은 상단 eyebrow(예: "Step 1"), `title`은 그 아래 본 제목(예: "캠페인 만들기")입니다.

```tsx
<Stepper variant="bar" current={1} steps={[
  { key: "campaign", label: "Step 1", title: "캠페인 만들기" },
  { key: "ad", label: "Step 2", title: "광고 만들기" },
  { key: "creative", label: "Step 3", title: "소재 만들기" },
]} />
```

막대 색은 토큰이 자동 결정합니다 — completed/current는 브랜드색, upcoming은 border. hex를 직접 지정하지 마세요.

## Props

| Prop      | 타입                       | 기본값       | 설명                                |
| --------- | -------------------------- | ------------ | ----------------------------------- |
| `steps`   | `StepItem[]`               | —            | 스텝 목록                           |
| `current` | `number`                   | —            | 현재 스텝 인덱스 (0-based)          |
| `variant` | `"numbered" \| "dots" \| "bar"` | `"numbered"` | 표시 변형 (`bar`=어드민 가로 막대) |

`<ol>` HTML 속성을 모두 지원합니다.

### StepItem

| 속성    | 타입        | 설명                                     |
| ------- | ----------- | ---------------------------------------- |
| `key`   | `string`    | 스텝 고유 키                             |
| `label` | `ReactNode` | 스텝 라벨 (없으면 라벨 슬롯 미렌더링). `bar`에서는 상단 eyebrow(예: "Step 1") |
| `title` | `ReactNode` | 보조 제목 — `bar` variant의 두 번째 라벨 줄(예: "캠페인 만들기"). numbered/dots에서는 무시 |

## data-slot 구조

```
ol [data-slot="root"] [data-variant]   (role="list")
└── li [data-slot="item"] [data-state="upcoming|current|completed"] [aria-current]
    ├── span [data-slot="indicator"] [data-variant]  (numbered면 번호/체크 / dots면 점)
    ├── span [data-slot="label"]                     (step.label이 있을 때)
    └── span [data-slot="connector"] [data-variant]  (마지막 스텝 제외)
```

`bar` 변형은 인디케이터/커넥터 대신 막대 + 2단 라벨 구조입니다:

```
└── li [data-slot="item"] [data-variant="bar"] [data-state]
    ├── span [data-slot="bar"] [data-variant="bar"]    (진행 막대)
    └── span [data-slot="label"] [data-variant="bar"]  (label/title이 있을 때)
        ├── span [data-slot="step"]                    (step.label — 상단 eyebrow)
        └── span [data-slot="title"]                   (step.title — 본 제목)
```

## 접근성

- `<ol role="list">` 시맨틱 구조
- 현재 단계 `<li>`에 `aria-current="step"` 부여
- `dots` 변형의 indicator는 `aria-hidden="true"` (장식적 점)
- connector는 `aria-hidden="true"` (장식적)
- 체크 SVG는 `aria-hidden="true"`
