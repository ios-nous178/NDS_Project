# Timeline

시간순 이벤트 로그 또는 단계 진행 상태를 보여주는 타임라인입니다. `items` 배열 + `mode`로 동작하며, 구 ActivityTimeline + StatusTimeline을 통합했습니다.

```tsx
import { Timeline } from "@nudge-design/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- 상담 예약 → 자가검사 → 결과 확인 같은 **시간순 이벤트 로그**를 보여줄 때 (`mode="activity"`)
- 정해진 단계의 진행 상태(완료/현재/예정)를 **트래커**로 보여줄 때 (`mode="tracker"`)
- 각 이벤트에 시점(날짜·시각), 제목, 설명, 상태 배지를 함께 표시해야 할 때
- 가로 단계 트래커가 필요할 때 (`mode="tracker"` + `direction="horizontal"`)

### 이럴 때 사용하지 마세요

- 단계 진척만 간결하게 보여줄 때 → Stepper를 사용하세요
- 다단계 폼 콘텐츠 + 네비게이션을 한 컨테이너로 묶을 때 → MultiStepForm을 사용하세요
- 항목이 20개 이상으로 길어질 때 → 페이지네이션/가상화를 함께 고려하세요

### 자주 하는 실수

- `items`의 각 항목은 `{ key, title, date?, description?, status?, statusLabel? }` 형태입니다. `title`은 필수(이벤트/단계 이름), `date`는 시점입니다.
- `mode="tracker"`는 `current`(0-based)로 상태를 파생합니다 — per-item `status`는 무시됩니다. 전부 완료는 `current={items.length}`로 줍니다.
- `mode="activity"`는 per-item `status`(`default` / `completed` / `ongoing` / `warning` / `error`)를 명시합니다. `statusLabel` 없이 `status`만 주면 점 색만 바뀌고 우측 배지는 뜨지 않습니다 — 둘 다 필요합니다.
- `direction="horizontal"`은 tracker에서만 의미가 있습니다(가로 단계 트래커). activity는 항상 세로입니다.
- `status="ongoing"`은 box-shadow ring 효과라 한 화면에 여럿 두면 시각 잡음이 됩니다. 보통 1개만 둡니다.
- 슬롯 자식(`nds-timeline-item`)·`steps` 속성은 폐기되었습니다 — `items` 배열 + `mode`를 사용하세요.

## 기본 사용법

```tsx
<Timeline
  mode="activity"
  items={[
    {
      key: "1",
      date: "2026.05.25",
      title: "상담 예약 완료",
      status: "completed",
      statusLabel: "완료",
    },
    {
      key: "2",
      date: "2026.05.28",
      title: "자가검사",
      status: "ongoing",
      statusLabel: "진행 중",
    },
  ]}
/>
```

## 모드 / 방향

| Prop | 값 | 설명 |
| ----------- | ----------------------------- | ------------------------------------------------------ |
| `mode` | `"activity"` / `"tracker"` | activity=시간순 이벤트 로그, tracker=단계 진행 트래커 |
| `direction` | `"vertical"` / `"horizontal"` | tracker 모드에서만 의미 (activity는 항상 세로) |

### activity 모드

시간순 이벤트 로그입니다. 항목마다 `status`로 점 색을, `statusLabel`로 제목 우측 배지를 지정합니다.

```tsx
<Timeline
  mode="activity"
  items={[
    { key: "a", date: "2026.05.25", title: "상담 예약 완료", status: "completed", statusLabel: "완료" },
    { key: "b", date: "2026.05.28", title: "자가검사", status: "ongoing", statusLabel: "진행 중" },
    { key: "c", date: "2026.06.01", title: "결과 검토", status: "warning", statusLabel: "확인 필요" },
  ]}
/>
```

상태 색은 `status` 값(`default` / `completed` / `ongoing` / `warning` / `error`)으로 자동 결정됩니다. hex를 직접 지정하지 마세요.

### tracker 모드

정해진 단계의 진행 트래커입니다. `current`(0-based) 인덱스로 각 단계 상태를 파생합니다 — `current` 미만은 `done`(체크), 같으면 `current`, 큰 인덱스는 `todo`입니다. per-item `status`는 무시됩니다.

```tsx
<Timeline
  mode="tracker"
  current={1}
  direction="horizontal"
  items={[
    { key: "r", title: "접수", date: "05/20" },
    { key: "p", title: "처리 중" },
    { key: "d", title: "완료" },
  ]}
/>
```

`direction="vertical"`(기본)이면 세로 단계 트래커, `direction="horizontal"`이면 가로 트래커로 좌우 half-line이 단계 사이를 잇습니다.

## CSS 변수

이 컴포넌트는 CSS 변수를 노출하지 않습니다. 색·간격은 `--semantic-*` 토큰으로 자동 결정됩니다(`--semantic-gap-comfortable`, `--semantic-bg-status-info`, `--semantic-inset-chip` 등).

## data-slot 구조

```
ol [data-slot="root"] [data-mode="activity|tracker"] [data-direction="vertical|horizontal"]   (role="list")
└── li [data-slot="item"] [data-mode] [data-state="done|current|todo"] [data-status] [aria-current]
    ├── div [data-slot="indicator"]
    │   ├── span [data-slot="dot"] [data-state] [data-status]   (tracker=번호/체크 / activity=점·아이콘)
    │   └── span.nds-timeline__line [data-state]                (마지막 항목 제외 / horizontal은 좌우 half-line, aria-hidden)
    └── div [data-slot="body"]
        ├── span [data-slot="date"]                             (item.date가 있을 때)
        ├── span [data-slot="title"] [data-state]
        │   └── span [data-slot="badge"] [data-status]          (item.statusLabel이 있을 때)
        └── p [data-slot="description"]                         (item.description이 있을 때)
```

---

## 접근성

- `<ol role="list">` 시맨틱 구조로 항목을 순서 있는 목록으로 노출합니다
- tracker 모드의 현재 단계 `<li>`에 `aria-current="step"` 부여
- 점(dot)·연결선(line)은 `aria-hidden="true"` (장식적 요소)
- tracker `done` 단계의 체크 SVG는 `aria-hidden="true"`
- 상태는 색뿐 아니라 `statusLabel` 배지(activity) / 번호·체크(tracker)로도 구분되므로 색에만 의존하지 않습니다
