# Tooltip

hover · focus 시 트리거 옆에 표시되는 짧은 보조 설명입니다. 4가지 위치(top/right/bottom/left)와 표시 지연을 지원합니다.

```tsx
import { Tooltip } from "@nudge-design/react";
```

---

## 사용 가이드

### 이럴 때 사용하세요

- 아이콘 버튼의 **이름 / 라벨 보조**가 필요할 때
- 짧은 안내(1~2줄)를 입력 필드 옆에 노출하고 싶을 때
- 키보드 포커스로도 확인 가능해야 할 때

### 이럴 때 사용하지 마세요

- 모바일 주 사용자 — hover 트리거가 없어 정보가 도달하지 않습니다. 본문 안내를 사용하세요
- 중요한 안내, 길고 구조화된 설명 → Popover / 본문 텍스트를 사용하세요
- 클릭/탭으로 열어야 하는 메뉴 → [DropdownMenu](/components/dropdown-menu)를 사용하세요

---

## 기본 사용

```tsx
<Tooltip content="이 검사는 약 5분 소요됩니다" placement="top">
  <IconButton aria-label="안내" icon={<InfoIcon />} />
</Tooltip>
```

`children`이 트리거 요소이며, 표시 위치는 `placement`로 지정합니다. `delay` 이후에 등장합니다.

## data-slot 구조

```
div [data-slot="root"]                       (mouseenter/leave + focus/blur로 노출 제어)
├── span [data-slot="trigger"]               (aria-describedby — 표시 중일 때)
└── div [data-slot="content"] [data-placement]   (role="tooltip")
    └── span (arrow)
```

표시 중일 때만 `[data-slot="content"]`가 DOM에 마운트됩니다. 컨텐츠는 `position: absolute; z-index: zIndex.popup`로 트리거 기준으로 배치됩니다.

## 접근성

- 표시 중인 툴팁에 `role="tooltip"` + 자동 생성 `id` 부여
- 트리거 `<span>`에 `aria-describedby={tooltipId}`로 연결 (표시 중일 때만)
- 마우스 enter/leave뿐 아니라 focus/blur로도 노출 — **키보드 사용자도 접근 가능**
- `pointer-events: none`으로 컨텐츠 위로 마우스가 가도 트리거 hover 상태 유지
