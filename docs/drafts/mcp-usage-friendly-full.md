## :sparkles: 작업을 시작하기 전, 30초만 점검해주세요

부탁을 드리기 전에 두 가지만 확인해주세요. 이 두 가지가 어긋나면 시안이 엉뚱한 폴더에 만들어지거나, AI 가 우리 DS 를 못 보고 그냥 코드만 만지게 됩니다.

### 1. 시안 폴더가 열려 있나요?

채팅창 하단 표시줄을 봐주세요.

#### 좋은 상태 — 시안 전용 폴더만 열림

![좋은 작업 폴더 — 시안 전용 폴더만 열린 상태. git 브랜치/워크트리 표시 없음](/img/install/workspace-good.png)

폴더 이름만 표시되고 git 브랜치 / 워크트리 같은 게 안 보이면 OK 입니다.

#### 나쁜 상태 — 시안 폴더가 아닌 다른 폴더가 열림

![나쁜 작업 폴더 — 다른 폴더가 열림. main 브랜치 + 워크트리 표시](/img/install/workspace-bad.png)

폴더 이름 옆에 git 브랜치(`main` 등)나 `워크트리` 표시가 보이시나요? 그렇다면 시안 폴더가 아니라 다른 작업 폴더가 열린 상태입니다. 거기서 시안을 만들면 그 폴더의 원본 코드와 섞일 수 있어요. 위쪽 메뉴의 **Open Folder (⌘ + O)** 로 시안 폴더를 다시 열어주세요.

### 2. 커넥터가 켜져 있나요?

채팅창의 ➕ 메뉴 → 커넥터 를 열어 `Nudge Design System` 토글이 ON 인지 확인해주세요.

![커넥터 확인 — 'Nudge Design System' 토글이 ON 상태](/img/install/connector-check.png)

또는 채팅창에 `/mcp` 를 입력하시면 목록에 `nudge-ds` 가 ✓ 로 보입니다. 그럼 OK 입니다.
(커넥터 UI 표시 이름 = `Nudge Design System`, 기술 ID = `nudge-ds` — 같은 것을 두 가지로 부릅니다.)

안 보이거나 꺼져 있으면 ⌘Q 로 완전 종료 후 재시작해주세요.

## :bulb: 꿀팁 — 권한 건너뛰기 모드

매번 "이 명령 실행해도 돼?" 를 묻는 것이 번거로우시다면, 모드를 한 번만 바꿔두시면 편하게 작업하실 수 있어요.

![Claude Code 모드 토글 — 권한 건너뛰기 선택됨](/img/install/skip-permission.png)

**켜는 방법:**

- 좌측 하단 모드 토글을 클릭 (또는 Shift + ⌘ + M)
- 메뉴에서 **권한 건너뛰기** 선택
- 좌측 하단에 `권한 건너뛰기` 뱃지가 떠 있으면 ON

⚠️ **시안 폴더에서만** 켜주세요. 다른 폴더에서 켜면 그 폴더의 진짜 소스가 자동 수정될 수 있습니다.

## :file_folder: 부탁드리기 전에 — 서비스 vs 어드민

만드시는 화면 종류에 따라 사용 라이브러리와 톤이 다릅니다. 부탁하실 때 반드시 명시해주세요.

| 종류                    | 예시                              | 라이브러리 | 톤                  |
| ----------------------- | --------------------------------- | ---------- | ------------------- |
| 서비스 (사용자 앱)      | 트로스트 / 지니어트 / NudgeEAP 앱 | Nudge DS   | 브랜드, 모바일 우선 |
| 어드민 (CMS / 백오피스) | 운영툴, 관리자 화면               | Ant Design | 운영툴, 데스크톱    |

- ❌ "회원 관리 페이지 만들어줘" — 종류가 모호해요
- ✅ "어드민 회원 관리 페이지"
- ✅ "트로스트 서비스 마이페이지"

종류만 명시해주시면 어떤 라이브러리·톤·규칙으로 만들지는 Claude 가 알아서 결정합니다. 다만 어드민 / 서비스를 한 화면에 섞지만 말아주세요.

## :speech_balloon: 매 턴마다 "넛지 DS 써서" 한 마디 붙여주기

수정 · 추가로 대화가 길어지면 Claude 가 MCP 호출을 잊고 그냥 코드만 만질 때가 있어요. 매번 한 마디만 끼워 넣어 주시면 카탈로그·가이드를 다시 참조해서 정확도가 올라갑니다.

- ❌ "여기 카드에 평점이랑 리뷰 수도 추가해줘."
- ✅ "넛지 DS 써서 여기 카드에 평점이랑 리뷰 수도 추가해줘. DS 에 별점 컴포넌트 있으면 그거 쓰고."

표현은 자유롭게 — 아래는 다 같은 MCP 를 가리킵니다.

- "넛지 DS 써서 ~" ← 가장 자연스러워요
- "Nudge DS 써서 ~" / "Nudge Design System 써서 ~" ← 커넥터 UI 이름
- "DS MCP 다시 확인하고 ~" / "DS 가이드 한 번 더 보고 ~"

첫 부탁에서는 CLAUDE.md 가 자동으로 MCP 를 호출해줍니다만, 여러 턴 이어지면 깜빡할 수 있어요. 수정 / 추가 / 검증 단계에서는 의식적으로 붙여주세요.

## :art: 시안을 만드는 흐름

큰 흐름은 6단계입니다. Claude 가 여러 단계를 묶어서 처리하기도 하니, 안 묶이면 한 단계씩 부탁해주세요.

1. **조사** — "이런 화면 만들 건데 어떤 DS 컴포넌트 / 아이콘 쓰면 좋아?"
2. **스캐폴드** — (시안 폴더 처음 쓸 때만) "이 폴더에 NudgeEAP 시안 프로젝트 세팅해줘"
3. **작성** — 아래 "부탁 잘 하는 공식" 대로 PRD + 5요소 정리해서 부탁
4. **검증** — "validate_mockup 으로 점검 + 위반 자동 수정"
5. **HTML 빌드 + 사용 리포트** — "넛지 DS 써서 HTML 한 파일 빌드 + report_mockup_usage"
6. **회고** — "DS 원칙에 맞는지 한 번 더 봐줘. 참고한 가이드도"

체크리스트: ☑ 조사 → ☑ 스캐폴드 → ☑ 작성 → ☑ 검증 → ☑ HTML + 리포트 → ☑ 회고

## :memo: 부탁을 잘 드리는 공식

좋은 부탁 = PRD + 최소 5요소.

### PRD — 기획자라면 꼭

기획 의도 / 사용자 시나리오 / 요구사항 / 우선순위를 정리해서 같이 던져주세요. PRD 가 있으면 Claude 가 의사결정(어떤 컴포넌트 / 어떤 톤 / 어떤 정보 위계)을 훨씬 정확히 잡아줍니다.

- 노션 / Figma PRD 문서가 있으시면 → 채팅창에 링크 붙이거나 핵심 부분을 텍스트로 복붙
- 짧으면 인라인으로 — "이 화면 목적은 사용자가 ~할 때 ~하기 위함. 1순위 액션은 ~"

### 최소 5요소

PRD 없이 빠르게 만드실 때도 아래 5개는 꼭 채워주세요.

| 요소          | 예시                                    |
| ------------- | --------------------------------------- |
| 종류          | 서비스 / 어드민                         |
| 브랜드·도메인 | 트로스트 / 지니어트 / NudgeEAP / —      |
| 페이지        | 상담사 목록 / 회원 관리                 |
| 기기          | 모바일 / 데스크톱                       |
| 핵심 구성     | 검색바·필터·카드·페이지네이션 (위→아래) |

예시:

> "트로스트 서비스 상담사 목록, 모바일. 상단 검색바 + 카테고리 필터(불안/우울/관계) + 상담사 카드 + 하단 페이지네이션."

> "어드민 회원 관리, 데스크톱, 운영툴 톤으로 조밀하게. 상단 검색·필터 + 회원 테이블(이름/이메일/가입일/상태/액션) + 우측 상단 '신규 등록' 버튼."

### 잘 부탁하는 6가지 팁

1. 매 턴마다 "넛지 DS 써서" 한 마디 — 수정·추가·검증 시 가장 중요
2. PRD 첨부 (있으면) — 의사결정 정확도 ↑
3. 종류 명시 — 서비스 vs 어드민
4. 브랜드 + 기기 명시
5. 핵심 구성은 위→아래 순서로
6. 한 번에 1~2개씩 수정 — 한꺼번에 5개 부탁 ❌

## :package: HTML 한 파일로 뽑기

DS MCP 의 `build_singlefile_html` 도구가 vite-plugin-singlefile 로 클릭 / 탭 인터랙션이 살아있는 한 파일을 만들어줍니다. 결과물 위치는 `dist/index.html` 입니다.

부탁하실 때 **"넛지 DS 써서 HTML 뽑아줘"** 라고 꼭 명시해주세요. 그래야 Claude 가 MCP 의 build 도구를 거치면서 vite 설정 / 라우팅 / 패키지 의존성을 알아서 맞춰줍니다. 그냥 "vite build" 같은 식으로 부탁하시면 설정이 안 맞아서 빈 화면이 나올 수 있어요.

- "넛지 DS 의 build_singlefile_html 써서 HTML 한 파일로 뽑아줘."
- "DS MCP 써서 HTML 빌드."

처음 한 번은 시안 폴더에 `vite-plugin-singlefile` 패키지가 자동 설치되고 vite 설정도 Claude 가 손봐줍니다. 두 번째부터는 빠릅니다.

정적 PNG / PDF 캡처 도구는 현재 없어요. 한 장면만 그림으로 필요하시면 브라우저에서 `dist/index.html` 열고 직접 스크린샷을 찍어주세요.

## :shield: Claude 가 알아서 잡아주는 것들

저장 시점에 자동으로 검증되고, 위반 사항이면 자동 수정 제안이 들어갑니다.

- hex / rgb → 토큰으로
- 인라인 픽셀 값 → 토큰 권장
- DS 에 없는 컴포넌트 / 아이콘 import → 차단
- 그라데이션 → DS 금지
- 잘못된 prop 값 (예: `size="md"` 오타) → 경고
- 이모지 / 기호 (😀 → ✓ ★ •) → 차단 후 정식 아이콘 안내
- 회색 버튼을 활성 CTA 로 사용 → 경고
- 여러 CTA 에 화살표 중복 → 1개만
- primary 버튼 남용 → 위계 조정
- Chip / Badge 장식 남용 → 상태 / 분류만
- 안내 영역 강조 중복 (배경 + 아이콘 + 배지 + 그라데이션) → 줄이기
- 서비스인데 antd 가 섞임 / 어드민인데 DS 가 섞임 → 자동 분리

함정이 자동 회피되는 핵심 컴포넌트:
`Button` · `Card` · `Chip` · `IconButton` · `Tabs` · `Select` · `Banner` · `Modal` · `List`

→ "이런 화면 만들어줘" 만 던져주시면 디테일은 Claude 가 알아서 처리해 드려요.
