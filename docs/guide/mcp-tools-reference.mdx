# MCP 도구 레퍼런스

`nudge-eap-ds` MCP 서버는 Claude 가 NudgeEAP 디자인시스템 정보를 정확하게 찾아 쓰도록 **21개 도구**를 제공합니다. 거의 모든 도구는 **Claude 에게 한국어로 부탁하면 자동 호출**되므로 직접 인자를 타이핑할 일은 없습니다. 이 페이지는 어떤 상황에 어떤 도구가 쓰이는지 한눈에 보여주는 레퍼런스입니다.

> 사용 흐름과 환경 설치는 [MCP 사용 가이드](/docs/NUDGE_EAP_DS_MCP_USAGE) 를 먼저 참고하세요.
> 단일 출처: `packages/mcp/src/tools/registry.ts` (도구 정의) + `packages/mcp/src/tools/guides.ts` / `setup.ts` (핸들러).

> 💡 **2025-Q2 통합 안내**
> `get_design_principles` / `get_dos_and_donts` / `get_admin_cms_guide` / `get_scope_advisory` / `get_inspector_setup` / `get_component_guide` / `get_pattern_guide` 는 **`get_guide({ topic })`** 로 통합됐습니다.
> `get_install_command` / `get_main_tsx_imports` / `get_update_instructions` / `create_claude_md` / `get_setup_instructions` 는 **`get_setup({ step })`** 로 통합됐습니다.
> `get_export_html_instructions` 는 액션 도구 **`build_singlefile_html`** 로 바뀌어, "안내" 가 아닌 "실제 빌드" 까지 한 번에 수행합니다.
> 옛 이름으로 호출하면 MCP 가 새 호출 형태를 안내하며 에러를 띄웁니다.

## 1. 브랜드 / 컴포넌트

작업을 시작하기 전 어떤 브랜드 / 어떤 컴포넌트를 쓸지 결정하는 도구들.

| 도구                | 용도                                                                                                | 호출 시점                            |
| ------------------- | --------------------------------------------------------------------------------------------------- | ------------------------------------ |
| `list_brands`       | brands/ 폴더 + 토큰 dist 에서 자동 발견된 서브 브랜드 목록(슬러그/이름/주색상/CSS 경로/ready 플래그) | 브랜드 분기 직전                     |
| `get_brand_info`    | 한 브랜드의 풀 정보 — 시멘틱 컬러, 폰트, css·js 임포트 경로, `main.tsx` import 순서                  | 특정 브랜드 작업 시작 시             |
| `list_components`   | 사용자 앱(Trost / Geniet / NudgeEAP) 용 DS React 컴포넌트 전체 목록. 어드민/CMS 화면에는 사용 금지   | 컴포넌트 선택 단계                   |
| `get_component`     | 정확한 컴포넌트명으로 props 상세 조회 (`name: "Button"` 처럼 case-sensitive)                          | 작성 전 props 확인                   |
| `search_component`  | 자연어 쿼리로 컴포넌트 검색 (`"tab"`, `"avatar"` 등)                                                  | 이름을 모를 때                       |

> ⚠️ 어드민/CMS 화면은 `list_components` 결과를 사용하지 말고 `get_guide({ topic: "admin-cms" })` 를 호출해 antd 5.5.1 규약을 따르세요.

## 2. 아이콘

| 도구          | 용도                                                                                                                     | 호출 시점       |
| ------------- | ------------------------------------------------------------------------------------------------------------------------ | --------------- |
| `list_icons`  | `@nudge-eap/icons` 전체 84개 + Figma Iconography(379:490) 카테고리 / Line·Filled·Color 스타일 메타데이터. `byCategory` 인덱스 포함 | 카탈로그 탐색  |
| `find_icon`   | 자연어 쿼리로 아이콘 검색 (`"search"`, `"arrow back"`). 결과에 category / style / pair 포함 — Line vs Filled 정확히 선택 가능 | 용도만 알 때    |

> 사이즈 / 터치 / 스타일 규칙은 `get_guide({ topic: "pattern:iconography" })`, 색상 토큰 매핑은 `get_guide({ topic: "pattern:icon-color" })` 로 보강하세요. 자세한 카탈로그는 [Docs · Icons](/docs/components/icons).

## 3. 토큰

| 도구             | 용도                                                                                                                                          | 호출 시점                       |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------- |

## 4. 가이드 (통합)

원칙·패턴·컴포넌트 가이드·어드민 규약·인스펙터 셋업·스코프 분기 안내까지 **단일 도구 `get_guide`** 로 통합돼 있습니다.

### `get_guide({ topic })`

`topic` 으로 무엇을 받을지 지정합니다.

| topic 패턴                  | 반환                                                                                                          | 호출 시점                                     |
| --------------------------- | ------------------------------------------------------------------------------------------------------------- | --------------------------------------------- |
| `principles`                | DESIGN.md 기반 — 브랜드 톤, 컬러 시멘틱, 타이포, 스페이싱, 엘리베이션, 셰이프, do's/don'ts, 금지 패턴             | 사용자 앱 목업 시작 시 **항상 호출**            |
| `dos-donts`                 | `principles` 의 do/don't 서브셋 — 짧은 sanity check 용                                                          | 목업 마무리 직전                              |
| `admin-cms`                 | 어드민 / CMS / 운영툴 / 백오피스 시안용 antd 5.5.1 규약 — sider, page header, search form, table, modal       | 어드민 시안 작업 시. 이때는 `principles` 대신 |
| `scope-advisory`            | 이 MCP 의 적용 범위 + 사용자 앱 vs 어드민 CMS 분기 규칙                                                         | 모호한 작업 진입 시                           |
| `inspector-setup`           | dev-only DsInspector 를 src/main.tsx 에 마운트하는 가이드 (Ctrl/Cmd+Shift+D 토글)                              | 인스펙터 수동 가이드가 필요할 때 (권장: 자동 — `get_setup({ step: "inspector" })`) |
| `component:<Name>`          | 큐레이션된 컴포넌트 가이드 — props 함정, 매트릭스, 접근성, Figma 노드. `Button` / `Card` / `Chip` / `IconButton` / `Tabs` / `Select` / `Banner` / `Modal` / `List` 등 | 핵심 컴포넌트 첫 사용 직전 — 항상 먼저 호출    |
| `pattern:<name>`            | UX 패턴 가이드 (아래 키 참고)                                                                                  | 위계 / 아이콘 / 밀도 결정이 모호할 때          |

**`pattern:` 키 목록**

`cta-group` · `icon-color` · `iconography` · `visual-reference` · `visual-antipatterns` · `notice` · `dropdown` · `dense-list`

### `list_figma_sync_status`

| 도구                       | 용도                                                                                          | 호출 시점                              |
| -------------------------- | --------------------------------------------------------------------------------------------- | -------------------------------------- |
| `list_figma_sync_status`   | 큐레이션 가이드별 Figma 노드 매핑 상태(figmaNodeUrl / sizeMatrix / stateMatrix)                  | 디자인 QA — Figma 정합 audit 누락 점검 |

## 5. 검증 / 집계

| 도구                  | 용도                                                                                                                                                                                                                                                        | 호출 시점                                                                                                              |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `validate_mockup`     | 목업 `.tsx` 소스(or `filePath`) 를 DS 규칙으로 검증 — 인라인 색/스페이싱, native 요소, 인라인 SVG, 이모지·텍스트 기호, gradient, **invalid-prop-value** (잘못된 union 리터럴, 예: `IconButton size="md"`), 패턴 안티패턴(CTA / Chip / Card / 아이콘 과다, antd 잔존 등) | 목업 작성 직후 / 저장 시                                                                                              |
| `suggest_replacement` | 인라인 컬러 / 스페이싱이 든 코드 스니펫 → 토큰 치환 제안                                                                                                                                                                                                       | `validate_mockup` 후 자동 fix 만들 때                                                                                  |
| `report_mockup_usage` | TSX 를 AST 로 파싱 → 각 JSX 를 `ds`(`@nudge-eap/react`) / `adminCms`(antd) / `customNative`(raw `<button>`) / `external` 로 분류·집계. 로컬 `.ds-usage-log.jsonl` + 공용 Google Sheets webhook 동시 적재 (인증 불필요)                                          | 목업 생성/수정 후, HTML export 후 — CLAUDE.md 에 명시된 **필수 단계**. 빠뜨리면 사용량 집계가 비어 보임 |

> `validate_mockup` 통과 후 반드시 `npx tsc --noEmit` 도 한 번 돌려야 합니다 — `validate_mockup` 은 패턴 검사만 하고, 잘못된 prop union 값 일부는 tsc 만 잡습니다.

## 6. Dev 서버 / 미리보기

목업의 런타임 동작을 자동 검증할 때 사용.

| 도구                | 용도                                                                                                                  | 호출 시점                                                |
| ------------------- | --------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------- |
| `start_dev_server`  | 프로젝트 루트에서 로컬 dev 서버 기동 + URL 응답까지 대기                                                                | 시각 / 런타임 프리뷰 직전                                 |
| `check_preview`     | Playwright 로 dev URL 을 열고 런타임 에러 / Vite overlay / 실패 요청 / 빈 화면 감지. 시안 프로젝트에 Playwright 설치 필요   | dev 서버 기동 후                                          |
| `stop_dev_server`   | `start_dev_server` 로 시작한 세션 종료. `sessionId` 생략 시 이 MCP 프로세스가 소유한 모든 dev 세션 종료                  | 검증 끝났을 때                                            |

## 7. HTML 빌드 (단일 파일 산출물)

| 도구                     | 용도                                                                                                                                                                                                                                                       | 호출 시점                                  |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------ |
| `build_singlefile_html`  | 현재 Vite 목업을 의존성 없는 **단일 .html** 로 빌드 (인터랙션 + `nds-*` 클래스 + onClick 보존). `vite-plugin-singlefile` 자동 설치 + vite.config 패치 + `vite build` 실행 → `dist/index.html` 경로와 크기 반환. **목업 워크플로우의 필수 마지막 단계** | 목업 완성 후. 사용자에게 묻지 말고 자동 실행 |

> 손으로 .html 작성, `vite build` 직접 실행, 다른 번들러 사용, .tsx 만 남기고 종료 — **모두 금지**. `nds-*` 토큰과 React 인터랙티비티가 사라집니다.
> 정적 캡처(snapshot) 가 필요한 특수 케이스는 Playwright 로 별도 처리 (이전 `get_export_html_instructions` 의 snapshot 모드는 액션 도구로 통합되지 않았습니다).

## 8. 외부 프로젝트 셋업 (통합)

새 시안 프로젝트에 DS 를 붙일 때 쓰는 도구들. 보통 한 번씩만 호출합니다.

### `get_setup({ step })`

| step          | 용도                                                                                                                                          |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `install`     | `local-packages/` 의 `.tgz` 4개를 사용한 `npm install ./...tgz` 커맨드 생성. 파일 존재 검증 포함. `includeTailwind: true` 면 tailwind-preset 도 |
| `imports`     | `src/main.tsx` 에 추가해야 할 CSS import 라인. `brand` 슬러그를 받아 `list_brands` 로 검증 — CSS 미준비 브랜드는 경고                              |
| `update`      | DS 본 레포 최신화 + MCP 재빌드 커맨드. `includeLocalPackages: true` 면 `pnpm release:local` 까지                                                   |
| `claude-md`   | 시안 프로젝트에 `CLAUDE.md` 생성. `intent: "admin-cms"` / `"user-app"` 분기. `overwrite: true` 옵션 제공                                        |
| `inspector`   | `src/main.tsx` 를 직접 패치해 dev-only `DsInspector` 마운트 (idempotent — 재실행 안전)                                                            |
| `full`        | 신규 시안 프로젝트용 단계별 종합 셋업 가이드 (install / imports / claude-md / withRouter / includeTailwind 옵션)                                  |

### `list_packages`

| 도구             | 용도                                                                                            | 호출 시점                          |
| ---------------- | ----------------------------------------------------------------------------------------------- | ---------------------------------- |
| `list_packages`  | DS 패키지 목록 — 버전, required / optional, 내부 의존성, peer deps, CSS 익스포트                 | 무엇을 install 해야 할지 파악할 때 |

## 호출 흐름 치트시트

### 새 사용자 앱 시안을 만들 때

```
1. get_guide({ topic: "scope-advisory" })            (분기 확인)
2. list_brands → get_brand_info                       (브랜드 선택)
3. get_guide({ topic: "principles" })                 (전역 규칙 로드)
4. (필요 시) list_components / search_component
5. (핵심 컴포넌트 첫 사용 직전) get_guide({ topic: "component:Button" })
6. (모호한 패턴이면) get_guide({ topic: "pattern:cta-group" })
7. (아이콘 필요) find_icon → list_icons
8. (토큰 매칭) lookup_token  /  suggest_replacement
9. validate_mockup                                    (저장 시)
10. npx tsc --noEmit                                  (prop union 검증)
11. start_dev_server → check_preview → stop_dev_server
12. build_singlefile_html                             (단일 HTML 산출 — 필수)
13. report_mockup_usage                               (집계 적재 — 필수)
```

### 어드민 CMS 시안을 만들 때

```
1. get_guide({ topic: "scope-advisory" })             (어드민이면 여기서 분기)
2. get_guide({ topic: "admin-cms" })                  (= antd 규약)
3. (목업 작성 — antd 우선)
4. validate_mockup → report_mockup_usage
```

### 새 시안 프로젝트 초기화

```
1. get_setup({ step: "full", intent: "user-app" | "admin-cms" })
2. list_packages → get_setup({ step: "install" })
3. (설치 후) get_setup({ step: "imports", brand: "trost" })
4. get_setup({ step: "claude-md" })
5. get_setup({ step: "inspector" })                   (dev-only DsInspector 자동 마운트)
6. start_dev_server → check_preview → stop_dev_server
```

### MCP 업데이트 확인

```
1. check_mcp_update                                   (새 .mcpb 있나?)
2. (있으면) get_setup({ step: "update", includeLocalPackages: true })
3. Claude Code 재시작
```

## 안 보일 때

- `/mcp` 입력 → `nudge-eap-ds` 가 떠 있는지 확인
- 떠 있는데 도구가 없으면 → Claude Code 재시작 (MCP 는 시작 시점에만 도구 로드)
- "Tool 'get_design_principles' has been consolidated. Use get_guide({ topic: 'principles' }) instead." 같은 에러가 보이면 → 옛 이름을 호출한 것. 위 통합 안내 표 참조
- 그래도 안 되면 → [MCP 사용 가이드 — 안 될 때](/docs/NUDGE_EAP_DS_MCP_USAGE)
