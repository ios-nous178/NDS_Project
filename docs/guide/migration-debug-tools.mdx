# 마이그레이션 & 디버깅 도구

디자인 시스템 마이그레이션 과정에서 **어떤 컴포넌트가 교체되었는지** 시각적으로 확인하고, **기존 화면과 비교**할 수 있는 개발 도구입니다.

---

## DS Highlight 디버깅 도구

마이그레이션된 페이지에서 NDS 컴포넌트가 적용된 영역을 점선 테두리 + 라벨로 표시합니다.

### 활성화 방법

| 방법              | 사용법                              | 설명                         |
| ----------------- | ----------------------------------- | ---------------------------- |
| **URL 파라미터**  | `?ds=1` 붙여서 접속                 | 즉시 활성화 + 토글 버튼 표시 |
| **키보드 단축키** | `Ctrl+Shift+D` (Mac: `Cmd+Shift+D`) | 모드 순환 + 토글 버튼 표시   |

기본 상태에서는 토글 버튼이 보이지 않습니다. 위 방법으로 활성화해야 표시됩니다.

### 보기 모드

토글 버튼 클릭 또는 단축키로 순환합니다.

| 모드     | 버튼 표시 | 설명                                                   |
| -------- | --------- | ------------------------------------------------------ |
| **OFF**  | `DS OFF`  | 하이라이트 없음                                        |
| **영역** | `DS 영역` | `DSMark`로 감싼 큰 영역만 표시 (파란 점선)             |
| **개별** | `DS 개별` | NDS 컴포넌트 하나하나 자동 감지하여 표시 (마젠타 점선) |
| **전체** | `DS 전체` | 영역 + 개별 동시 표시                                  |

### 자동 컴포넌트 감지

NDS 컴포넌트는 모두 `nds-{name}__root` 패턴의 CSS 클래스를 가지고 있습니다. 디버깅 도구는 이 패턴을 자동으로 감지하여 컴포넌트 이름을 추출합니다.

```
nds-checkbox__root  → Checkbox
nds-search-input__root → SearchInput
nds-select__root → Select
nds-card__root → Card
```

**새 NDS 컴포넌트를 추가하면 별도 등록 없이 자동으로 감지됩니다.**

### 토글 버튼 드래그

토글 버튼은 드래그하여 원하는 위치로 이동할 수 있습니다. 다른 개발 도구(React DevTools, vConsole 등)와 겹치는 경우 위치를 옮기세요.

- **클릭**: 모드 순환
- **드래그**: 버튼 위치 이동

### 구현 원리

```
┌──────────────────────────────────────────────────┐
│ document.body                                     │
│                                                   │
│  ┌─ 페이지 DOM ──────────────────────────────┐   │
│  │  ┌─ overflow: hidden 부모 ──────────────┐ │   │
│  │  │  ┌─ NDS 컴포넌트 ─┐                  │ │   │
│  │  │  │  Checkbox       │  ← getBoundingClientRect() 측정
│  │  │  └─────────────────┘                  │ │   │
│  │  └───────────────────────────────────────┘ │   │
│  └────────────────────────────────────────────┘   │
│                                                   │
│  ┌─ Portal Overlay (position: absolute) ─────┐   │
│  │  ┌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┐                   │   │
│  │  ╎ Checkbox            ╎  ← 점선 테두리 + 라벨│
│  │  └╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌┘                   │   │
│  └────────────────────────────────────────────┘   │
│                                                   │
│  ┌─ 토글 버튼 (position: fixed, draggable) ──┐   │
│  │  DS 전체                                   │   │
│  └────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────┘
```

Portal 오버레이는 DOM 트리 바깥에서 렌더링되므로 **어떤 `overflow: hidden/auto` 부모에서도 잘리지 않습니다.**

### 패키지 구조

```
@nudge-design/react
├── DSHighlightProvider   — Provider (단축키/URL 감지, 모드 관리)
├── DSMark                — 영역 등록 래퍼
├── useDSHighlight        — 모드/cycle 접근 훅
└── (DSOverlayPortal)     — Portal 오버레이 (내부)
    └── (DSHighlightToggle) — 드래그 가능 토글 버튼 (내부)
```

> 실제 구현은 `@nudge-design/react` 패키지 내부에 있으므로, 서비스 프로젝트에서는 별도 구현 없이 import만 하면 됩니다.

| 경로                         | 설명           |
| ---------------------------- | -------------- |
| `/service/search/partner`    | 기존 페이지    |
| `/service/search/partner-ds` | DS 적용 페이지 |

### 교체된 컴포넌트 목록

| 영역          | 기존 컴포넌트                                  | NDS 컴포넌트                          |
| ------------- | ---------------------------------------------- | ------------------------------------- |
| 검색 입력     | `PartnerSearchInput` (Tailwind + Image 아이콘) | `SearchInput` (filled variant)        |
| 필터 칩       | `ActiveFilterChip` (Tailwind div)              | `Chip` (outlined, selected, onRemove) |
| 정렬 드롭다운 | `PartnerSortDropdown` (직접 구현)              | `Select`                              |
| 필터 체크박스 | `Checkbox` (이미지 기반)                       | `Checkbox` (NDS 커스텀 인디케이터)    |
| 지역 드롭다운 | `FilterSelect` (직접 구현 Portal)              | `Select`                              |
| 상담사 카드   | `PartnerCard` (Tailwind article)               | `Card.Root` (outlined, clickable)     |
| 키워드 태그   | `li` (인라인 스타일)                           | `Chip` (soft, sm)                     |
| 빈 결과       | `Image + p` (직접 구현)                        | `EmptyState`                          |

## 코드 마이그레이션 예시

실제 `homepage`의 상담사 검색 페이지(`partner-ds`)에서 적용된 코드 변경 사례입니다.

### 1. 상단 선택 필터 (Active Filter)

기존의 복잡한 Tailwind 조합과 커스텀 SVG 아이콘을 NDS `Chip` 컴포넌트 하나로 대체했습니다.

<Tabs defaultValue="after" values={[
  { label: 'Before (기존)', value: 'before' },
  { label: 'After (NDS 적용)', value: 'after' }
]}>
  <TabItem value="before">

```tsx
// ActiveFilterChip.tsx (직접 구현)
<div className="inline-flex items-center gap-1 pl-[14px] pr-[10px] py-[8px] rounded-lg border border-[#2b96ed] text-[#2b96ed] text-sm">
  <span>{label}</span>
  <div onClick={onRemove} className="cursor-pointer">
    <Image src="/ic-close-blue.svg" width={16} height={16} />
  </div>
</div>
```

  </TabItem>
  <TabItem value="after">

```tsx
// NDS Chip 사용
<Chip
  label={filter.itemLabel}
  variant="outlined"
  selected
  size="md"
  className="px-[14px]"
  onRemove={() => handleRemoveFilter(...)}
/>
```

  </TabItem>
</Tabs>

### 2. 카드 내 키워드 태그

기존 `li` 태그와 인라인 스타일 대신 NDS `Chip`의 `ghost` 변형을 사용하고, Tailwind 클래스로 미세한 여백을 조정했습니다.

<Tabs defaultValue="after" values={[
  { label: 'Before (기존)', value: 'before' },
  { label: 'After (NDS 적용)', value: 'after' }
]}>
  <TabItem value="before">

```tsx
// 기존 li 기반 태그
<li className="px-2 py-1 bg-[#f1f8fd] text-[#2b96ed] text-[13px] rounded-md">#{topic.label}</li>
```

  </TabItem>
  <TabItem value="after">

```tsx
// NDS Chip (ghost) + Tailwind px 오버라이드
<Chip label={`#${topic.label}`} variant="ghost" size="sm" className="px-2" />
```

  </TabItem>
</Tabs>

커스텀 컴포넌트(`PartnerCardTypeTag`) 내부의 복잡한 스타일링을 NDS `Chip`의 `icon` 프롭과 `ghost` 변형으로 표준화했습니다.

<Tabs defaultValue="after" values={[

  </TabItem>
  <TabItem value="after">

```tsx

  size="sm"
```

  </TabItem>
</Tabs>

### 4. 검색 입력창 (SearchInput)

기존의 직접 구현된 `input` 태그, 이미지 아이콘, 클리어 버튼 상태 관리 로직을 NDS `SearchInput`으로 대체하여 코드를 획기적으로 줄였습니다.

<Tabs defaultValue="after" values={[
  { label: 'Before (기존)', value: 'before' },
  { label: 'After (NDS 적용)', value: 'after' }
]}>
  <TabItem value="before">

```tsx
// PartnerSearchInput.tsx (상태 관리 + 이벤트 핸들러 직접 구현)
const [value, setValue] = useState(defaultValue);
const handleKeyPress = (e) => {
  if (e.key === "Enter") onSearch(value);
};

<div className="relative w-full">
  <input
    value={value}
    onChange={(e) => setValue(e.target.value)}
    onKeyDown={handleKeyPress}
    className="w-full h-12 pl-4 pr-10 border rounded-lg"
  />
  <Image src="/ic-search.svg" className="absolute right-3 top-3" />
  {value && (
    <button onClick={() => setValue("")} className="...">
      X
    </button>
  )}
</div>;
```

  </TabItem>
  <TabItem value="after">

```tsx
// NDS SearchInput (내장 로직 활용)
<SearchInput
  value={value}
  onChange={(e) => setValue(e.target.value)}
  onSearch={() => onSearch(value)}
  onClear={() => onSearch("")}
  placeholder="상담사 이름 또는 키워드 검색"
  variant="filled"
/>
```

  </TabItem>
</Tabs>

### 5. 정렬 드롭다운 (Select)

`useState`, `useRef`, 외부 클릭 감지 로직 등으로 복잡했던 직접 구현 드롭다운을 NDS `Select` 컴포넌트 하나로 통합했습니다. (포털 렌더링 포함)

<Tabs defaultValue="after" values={[
  { label: 'Before (기존)', value: 'before' },
  { label: 'After (NDS 적용)', value: 'after' }
]}>
  <TabItem value="before">

```tsx
// PartnerSortDropdown.tsx (복잡한 UI 로직 포함)
const [isOpen, setIsOpen] = useState(false);
const dropdownRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  const handleClickOutside = (e) => {
    if (dropdownRef.current && !dropdownRef.current.contains(e.target)) setIsOpen(false);
  };
  document.addEventListener("mousedown", handleClickOutside);
  return () => document.removeEventListener("mousedown", handleClickOutside);
}, []);

<div ref={dropdownRef} className="relative">
  <button onClick={() => setIsOpen(!isOpen)}>{label}</button>
  {isOpen && <ul className="absolute ...">...</ul>}
</div>;
```

  </TabItem>
  <TabItem value="after">

```tsx
// NDS Select (완성된 UI 컴포넌트)
<Select
  options={options}
  value={value}
  onValueChange={onChange}
  fullWidth={false}
  className="w-[120px]"
/>
```

  </TabItem>
</Tabs>

## 미디어 파일 추가 방법

### 스크린샷

1. 스크린샷을 `apps/docs/static/img/migration/` 에 저장

### 영상

1. 영상을 `apps/docs/static/video/migration/` 에 저장
2. 이 문서에서 해당 `{/* */}` 주석을 해제
3. `<source src="/video/migration/파일명.mp4" />` 경로 확인

권장 형식:

- 스크린샷: PNG (2x 해상도 권장)
- 영상: MP4 (H.264, 10MB 이하 권장)
- 파일명: kebab-case (`partner-before-pc.png`)

## 설치 및 사용

`DSHighlightProvider`, `DSMark`, `useDSHighlight`는 **`@nudge-design/react` 패키지에 포함**되어 있습니다.

### 1. CSS import (1회)

```tsx
// layout.tsx 또는 page.tsx
import "@nudge-design/react/styles.css";
```

### 2. Next.js App Router에서 사용

Next.js에서는 `'use client'` 지시어가 필요하므로 re-export 래퍼를 만듭니다.

```tsx
// src/components/DSHighlight.tsx
"use client";

export { DSHighlightProvider, DSMark, useDSHighlight } from "@nudge-design/react";
```

### 3. 페이지에 적용

```tsx
import { DSHighlightProvider, DSMark } from "@/components/DSHighlight";

export default function MyPageDS() {
  return (
    <DSHighlightProvider>
      {/* 영역 단위 마크 (선택) */}
      <DSMark label="SearchInput">
        <SearchInput placeholder="검색" />
      </DSMark>

      <DSMark label="Card + Chip">
        <CardList />
      </DSMark>

      {/* 개별 NDS 컴포넌트는 자동 감지 — 별도 마크 불필요 */}
    </DSHighlightProvider>
  );
}
```

### 4. Vite/CRA 등 다른 환경

`'use client'`가 불필요하므로 직접 import합니다.

