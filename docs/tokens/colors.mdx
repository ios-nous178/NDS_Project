# 색상 토큰 (Raw Palette)

Nudge Design System 의 **원시(raw) 색상 팔레트** 입니다. Figma Design Guide(`430:4212`) 실측을 SSOT 로 정의합니다.

```tsx
import { colors } from "@nudge-design/tokens";

colors.blue[500]; // → "#2B96ED"
colors.neutral["00"]; // → "#FFFFFF" (white — string 키 주의)
```

:::tip 시멘틱 토큰은 별도 페이지
이 페이지는 **팔레트 원본(primitives)** 만 다룹니다.
실제 UI 에서는 **`--nds-*` / `--color-semantic-*` 같은 역할 기반 토큰**을 우선 사용하세요 — [시멘틱 토큰](/semantic-tokens) 페이지 참고.
:::

아래 카탈로그는 `@nudge-design/tokens` 의 `colors` 객체를 그대로 렌더합니다.
swatch 클릭 시 토큰 경로(`colors.blue[500]` 등)가 클립보드에 복사됩니다 — 코드에 바로 붙여넣어 쓰면 토큰 추적이 끊기지 않습니다.

## 사용 원칙

1. **하드코딩한 hex 대신 토큰 경로로 참조** — `#2B96ED` 박지 말고 `colors.blue[500]` 또는 시멘틱 변수
2. **컴포넌트 안에서는 시멘틱 우선** — `colors.blue[500]` 도 가능하면 `var(--nds-text-brand)` 처럼 의도가 드러나는 변수로
3. **brand 별 색감 차이**는 시멘틱 변수가 흡수 — primary 색이 Trost/Geniet/NudgeEAP 마다 다른 경우, 컴포넌트는 시멘틱만 참조하고 brand override 가 raw 값을 주입
4. **명도(luminance) 기반 텍스트 색** — 위 카탈로그가 자동으로 light/dark chip text 결정 (가독성 임계 0.55). 외부에서 같은 swatch UI 만들 땐 같은 방식으로

---

## 사용 예시

```tsx
import { colors } from "@nudge-design/tokens";

// raw 참조 — 시멘틱이 없을 때만
const inkSpot = { background: colors.neutral[900] };

// 시멘틱 (권장) — 의도 + brand-aware
const brandText = { color: "var(--nds-text-brand)" };
const surface = { background: "var(--nds-bg-surface)" };
```

## 새 색상 추가하기

`packages/tokens/src/colors.ts` 가 SSOT — 직접 수정하지 말고 `DESIGN.md` 의 YAML 정의에 색을 먼저 추가한 뒤 `pnpm generate:tokens` 으로 재생성하세요.
