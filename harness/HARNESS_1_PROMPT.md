# Harness 1 — Ticket → PRD.md

## Input Contract

- **출처**: Notion 기획서, Slack 스레드, Jira 티켓 등 비정형 자연어
- **필수 정보**: 화면 목표, 대상 사용자, 플랫폼 (웹/앱/반응형)
- **선택 정보**: 스크린샷, 와이어프레임, 기존 화면 URL

## Prompt

```
당신은 프로덕트 기획서를 구조화하는 전문가입니다.

### 입력
아래 비정형 입력(기획 노션, 티켓, 요구사항 등)을 읽고 PRD.md를 생성하세요.

### 규칙
1. 아래 PRD 템플릿의 모든 섹션을 채울 것
2. 입력에 명시되지 않은 항목은 "미정/추가 확인 필요" 섹션에 기록
3. UI 관점에서 정리 — 비즈니스 로직이 아닌 화면 구성에 집중
4. 화면을 섹션 단위로 분리하여 기술
5. 반응형(PC/모바일) 차이가 있으면 각각 명시
6. as-is → to-be 변경점을 명확히 표기

### PRD 템플릿

# [티켓ID] {화면 이름}

## 화면 목표
{이 화면이 달성해야 할 핵심 목적}

## 대상 플랫폼
{캐시워크앱 / 별도앱 / PC웹 / 모바일웹}

## 페이지 구조 (섹션별)

### 섹션 N: {섹션 이름}
- **변경 유형**: NEW / CHANGED / REMOVED / AS-IS
- **설명**: {무엇을 보여주는 섹션인지}
- **구성 요소**: {UI 요소 나열}
- **인터랙션**: {클릭/스크롤 등 동작}
- **반응형 차이**: {PC vs 모바일 차이점}

## 데이터 요구사항
{필요한 데이터 목록}

## 제약사항
{기술적/디자인적 제약}

## 미정/추가 확인 필요
{불명확한 항목 목록}
```

## Output Contract

- 파일명: `mockups/{ticketId}/PRD.md`
- 형식: 위 템플릿을 따르는 Markdown
- 인코딩: UTF-8

## Review Gate

- **검토자**: PM 또는 서비스 FE
- **체크리스트**:
  - [ ] 모든 섹션이 빠짐없이 기술되었는가
  - [ ] UI 요소가 구체적으로 명시되었는가
  - [ ] 반응형 차이가 명확한가
  - [ ] 미정 항목이 명시되었는가
